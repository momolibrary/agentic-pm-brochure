## 验收标准

### 必须回答的问题

1. **为什么不是所有请求都适合"直接调接口"？消息队列解决什么问题？**
   - 期望回答：直接调接口是同步的，高峰期所有请求同时涌入会把系统压垮。消息队列让请求排队处理（削峰），且发送方不用等接收方处理完（解耦）。
2. **出了 Bug 为什么开发第一反应是"看日志"？日志和监控有什么区别？**
   - 期望回答：日志记录系统运行的每一步细节（黑匣子），监控实时观察系统健康状况（心电图）。看日志能追溯问题发生的具体过程，监控能发现系统整体是否异常。
3. **API 网关为什么像"小区门卫"？为什么不让外部直接访问后端？**
   - 期望回答：API 网关是所有外部请求的统一入口，负责验证身份、控制流量、记录日志。不让外部直接访问后端是为了安全（统一鉴权）和控制（统一限流）。

### 读者能做到的事

1. **能理解消息队列的异步通信场景**
   - 验证方式：能判断一个业务场景是否适合用消息队列
2. **能理解日志和监控系统的作用**
   - 验证方式：能解释为什么系统需要日志和监控
3. **能理解 API 网关的职责**
   - 验证方式：能列出 API 网关的主要功能

### 必须纠正的误解

1. **误解："直接调接口最快，消息队列多此一举"**
   - 正确理解：同步调用在高并发时会压垮系统，消息队列用异步排队保证系统稳定。
2. **误解："线上出问题就是代码写错了"**
   - 正确理解：很多线上问题需要日志/监控数据才能定位，不是"看代码"就能解决的。

### 必须准确的概念

| 术语 | 定义 | 本章使用场景 |
|------|------|-------------|
| **消息队列** | 系统间异步通信的管道（如 Kafka、RabbitMQ） | A8-09 核心概念 |
| **日志** | 记录系统运行过程中每一步发生的事的记录 | A8-10 核心概念 |
| **API 网关** | 所有外部请求进入系统的统一入口 | A8-11 核心概念 |
