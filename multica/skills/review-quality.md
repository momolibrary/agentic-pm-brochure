# Skill: review-quality

质量审核验收能力

## 功能

基于 TDD 验收标准执行章节审核：
- 测试用例通过检查
- 章节结构验收
- 文风验收
- 综合质量评估

## 输入

- 章节内容文件
- TDD 验收标准文档
- lint 检查报告

## 输出

- 验收报告
- 通过/不通过判定
- 修订建议

## 执行流程

1. 加载验收标准
2. 检查问题回答情况
3. 检查能力传达情况
4. 检查误解纠正情况
5. 检查术语准确性
6. 检查结构完整性（10 要素 + 五段节奏映射）
7. **检查章节节奏**：全章字数、各段字数预算、总结是否不超 50 字
8. **执行 So What 测试**：逐段检查信息增量，标记无增量段落
9. 检查文风合规性
10. 生成验收报告

## 验收维度

| 维度 | 检查项 |
|------|--------|
| 测试用例 | Q1-Q3 回答准确 |
| 能力传达 | A1-A3 可执行 |
| 误解纠正 | M1-M2 正确理解 |
| 术语准确 | 与术语表一致 |
| 结构完整 | 10要素齐全，映射进五段节奏 |
| 章节节奏 | 全章 2500-3000 字，各段字数在预算内（见 style-guide.md 五段节奏表） |
| 信息增量 | 每段通过 So What 测试——无新信息的段落必须删除 |
| 文风合规 | 符合文风指南 |

## 质量标准

- 通过率必须 100%
- 所有检查项必须通过
- 问题必须有修复建议
