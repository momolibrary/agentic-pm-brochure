# Skill: chapter-draft

章节初稿写作能力

## 功能

根据验收标准（TDD）写作章节初稿，确保：
- 回答所有必须回答的问题
- 传达读者应获得的能力
- 纠正常见误解
- 使用准确术语

## 输入

- 验收标准文档（test_cases）
- 术语表引用
- 文风指南引用
- 素材需求（案例、类比、图示）

## 输出

- 章节初稿 Markdown 文件
- 包含完整10要素结构

## 执行流程

1. 解析验收标准，提取测试用例
2. 检查术语表，确认关键术语定义
3. 检查素材库，获取案例和类比
4. **按五段节奏公式规划章节结构**（痛点场景 ~300 字 → 核心概念 ~800 字 → 开发怎么看 ~400 字 → 怎么做 ~600 字 → 一句话总结 ~50 字），将 10 要素映射进 5 个大段
5. 按规划写作，严格控制每段字数预算
6. 确保每个测试用例被覆盖
7. **执行信息增量契约**：逐段 So What 测试——"这一段相比上一段，读者多知道了什么？"答案为"没有"则删掉
8. 执行自查清单（含节奏自检清单）

## 质量标准

- 所有测试用例 100% 覆盖
- 10要素结构完整，映射进五段节奏
- 全章字数 2500-3000 字，各段字数在预算 ±容差 内
- 每段通过 So What 测试（信息增量契约）
- 文风符合指南（短句、适度幽默、无空洞表达）
- 术语使用一致

## 依赖

- `term-lookup`：术语查询（运行时调用）
- 案例素材：由 [@案例师](mention://agent/2eb4c3b6-d91c-4372-9245-61769ab1032b) 在写作阶段前准备，存放于 `assets/cases/` 或 `manuscript/cases/`（非运行时依赖，通过工作流时序保证）
