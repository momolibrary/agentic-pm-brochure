## 类比一：厨房做菜 — 食材精准备料

### 类比场景

老李对小想说：

"你想想，厨师做菜前要备料。新手厨师把整个菜市场的东西都搬进厨房，结果呢？食材堆满台面，找不到要用的东西，做出来的菜味道混杂。老厨师呢？只拿今天要做的菜品需要的食材，台面清爽，动作利索。"

### 对应概念映射

| 厨房场景 | AI 上下文管理 |
|---------|--------------|
| 整个菜市场 | 整个代码仓库 |
| 搬进厨房的所有食材 | 把所有文件塞给 AI |
| 台面堆满找不到东西 | 上下文过载，AI 找不到关键信息 |
| 只拿需要的食材 | 精准挂载目标文件夹 |
| 台面清爽动作利索 | 上下文干净，AI 输出质量高 |

### 类比要点

- **误区**：以为食材越多越好，结果越做越乱
- **正确做法**：按菜品需求精准备料
- **产品经理理解点**：备料和喂料一样，贪多嚼不烂

## 类比二：图书馆查资料 — 书架定位

### 类比场景

小想抱怨："我让 AI 帮我改代码，它改得乱七八糟，根本不懂我的意思。"

老李问："你给了它什么？"

小想："整个项目文件夹，大概两百多个文件。"

老李笑了："你去图书馆查资料，是把整个图书馆的书都搬到桌上，还是只去相关书架找？"

小想恍然大悟。

### 对应概念映射

| 图书馆场景 | AI 上下文管理 |
|----------|--------------|
| 整个图书馆的书 | 整个项目仓库 |
| 把所有书搬到桌上 | 把所有文件塞给 AI |
| 只去相关书架 | 精准挂载相关文件夹 |
| 排除借阅区、仓库书 | 排除 .git、node_modules |
| 多开几张桌查不同主题 | 多开 Chat 平行探索 |

### 类比要点

- **噪音排除**：借阅区、仓库的书不是你现在要查的，AI 的 node_modules、.git 也不是
- **平行探索**：查多个主题开多个桌，改多个功能开多个 Chat
- **择优合并**：每个桌查到的信息汇总，每个 Chat 的输出择优合并

## 类比三：新员工入职 — 信息喂养节奏

### 类比场景

老李说："AI 就像新入职的员工。你第一天就把公司十年所有文档、所有代码、所有会议记录堆给他，他只会崩溃。好经理怎么带新人？第一天给核心架构图，第二天给当前项目文档，第三天才看历史记录。"

### 对应概念映射

| 新员工入职 | AI 上下文管理 |
|-----------|--------------|
| 十年所有文档 | 整个仓库历史 |
| 第一天堆满崩溃 | 上下文过载质量下降 |
| 分天按需喂给 | 分阶段精准挂载 |
| 核心架构图先给 | 当前任务核心文件先挂 |
| 历史记录后看 | 历史代码必要时再查 |

### 类比要点

- **节奏喂养**：不是一次塞完，是按任务节奏喂
- **核心优先**：当前任务相关文件先挂，历史文件必要时才查
- **信息焦虑**：新人被信息淹没会焦虑，AI 被上下文过载会"幻觉"

## 类比四：手术台 — 无菌环境

### 类比场景

"AI 做代码手术，就像医生做外科手术。手术台必须干净，不能堆满杂物。node_modules 就像手术台旁边的垃圾桶，把它留在台上干嘛？挡视线、占地盘、还可能碰倒。"

### 对应概念映射

| 手术场景 | AI 上下文管理 |
|---------|--------------|
| 手术台 | AI 的上下文窗口 |
| 堆满杂物 | 塞满无关文件 |
| 垃圾桶在旁边 | node_modules、.git 等噪音 |
| 干净手术台 | 精准挂载，排除噪音 |
| 医生专注手术区 | AI 专注当前任务区 |

### 类比要点

- **噪音就是风险**：杂物可能碰倒干扰手术，噪音可能干扰 AI 输出
- **干净才能精准**：手术台干净医生才专注，上下文干净 AI 才精准
- **排除意识**：好医生会清理台面，好 PM 会排除噪音

## 类比五：多窗口办公 — 平行探索

### 类比场景

"你写方案要参考三个案例，是开一个窗口来回切换，还是开三个窗口并列对比？肯定是后者。AI 的 Chat 也一样，改三个功能，开三个 Chat，每个 Chat 只挂载那个功能相关的文件。"

### 对应概念映射

| 多窗口办公 | 多开 Chat 平行探索 |
|----------|------------------|
| 一个窗口来回切换 | 一个 Chat 处理多任务 |
| 信息混杂容易忘 | 上下文混杂容易出错 |
| 开三个窗口并列 | 开三个 Chat 平行 |
| 每窗口看一个案例 | 每个 Chat 改一个功能 |
| 最后汇总对比 | 输出择优合并 |

### 类比要点

- **隔离原则**：一个窗口一个案例，一个 Chat 一个功能
- **对比择优**：多窗口对比选最优，多 Chat 输出选最优
- **合并策略**：各窗口信息汇总，各 Chat 输出合并
