## 故事一：把整个项目塞给 AI 的翻车现场

### 场景设定

- **人物**：小想（AI 原生代实习生）、老李（产品总监）
- **背景**：公司新上线 Claude Code，小想迫不及待用 AI 改一个登录功能
- **时间**：周五下午

### 故事正文

小想在工位上敲键盘，一脸兴奋。老李路过，问："在干嘛呢？"

"用 AI 改登录模块！" 小想指着屏幕上的 Chat 界面。

老李看了一眼，眉头皱起："你把整个项目文件夹都塞给 AI 了？"

小想点头："是啊，让它先了解整个项目，改起来更准确嘛。"

老李叹气："两百多个文件，上下文塞满了。你看看 AI 输出的代码。"

小想一愣，滚动屏幕看 AI 给出的修改建议。她发现问题了：AI 改的代码风格一会儿是这个文件的风格，一会儿是那个文件的风格；改的逻辑和另一个文件的逻辑混淆了；甚至还引用了一个根本不存在的方法。

"这……怎么这么乱？" 小想困惑。

老李坐下解释："AI 的上下文窗口就像你的脑子。你一下子塞两百多本书，脑子会乱。AI也一样，塞太多，它会'幻觉'——把不同文件的代码风格混在一起，还可能编造不存在的东西。"

小想问："那我该怎么喂？"

老李说："精准挂载。登录功能涉及哪些文件？auth 目录、login 页面、用户状态管理。只挂这几个文件夹，其他的不挂。就像你做一道菜，只拿那道菜需要的食材。"

### 故事要点

| 要点 | 教训 |
|------|------|
| 贪多嚼不烂 | 把整个项目塞给 AI，上下文过载 |
| 幻觉现象 | AI 会混淆不同文件的代码风格 |
| 精准挂载 | 只挂当前任务相关的文件夹 |
| 类比强化 | 像做菜只拿需要的食材 |

### 对话金句

老李："上下文窗口就像你的脑子。你一下子塞两百多本书，脑子会乱。"

小想："我以为越多越好……"

老李："备料和喂料一样，贪多嚼不烂。"

## 故事二：node_modules 没排除的惨案

### 场景设定

- **人物**：小想、老李
- **背景**：小想让 AI 分析一个性能问题，AI 给出的建议完全偏题
- **时间**：周一上午

### 故事正文

小想拿着打印的 AI 分析报告找到老李："李总，AI 说我们的性能瓶颈在 lodash 库里，建议换库。但我觉得不对……"

老李接过报告扫了一眼，笑了："AI 把 node_modules 也吃进去了。"

"啊？" 小想一愣。

老李指着报告："你看，AI 分析的是 lodash 的源码，不是你的业务代码。它被 node_modules 里的几万行代码带跑了方向。你的真正性能问题在业务逻辑的循环嵌套，AI 根本没看到。"

小想懊恼："我把整个项目目录挂了……"

老李说："下次记住，node_modules、.git、build、dist 这些目录要排除。就像医生做手术，手术台旁边不能堆垃圾桶，否则医生眼睛被垃圾桶挡住，看不清手术区域。"

小想问："怎么排除？"

老李："用 .claudeignore 文件，和 .gitignore 语法一样。写上 node_modules/、.git/、dist/、build/，这些就不进上下文了。"

### 故事要点

| 要点 | 教训 |
|------|------|
| 噪音干扰 | node_modules 被吃进去，AI 分析偏题 |
| 分析方向被带跑 | AI 看的是噪音文件，不是目标文件 |
| 排除工具 | .claudeignore 排除噪音目录 |
| 手术台类比 | 噪音目录像垃圾桶挡视线 |

### 对话金句

老李："node_modules、.git、build、dist 这些目录要排除。就像医生做手术，手术台旁边不能堆垃圾桶。"

小想："我把整个项目目录挂了……"

老李："AI 被几万行代码带跑了方向，真正的问题根本没看到。"

## 故事三：多开 Chat 平行探索的成功案例

### 场景设定

- **人物**：小想、老李
- **背景**：小想要同时改三个功能模块，用多开 Chat 策略成功
- **时间**：周三下午评审会前

### 故事正文

周三下午有评审会，小想要在会前完成三个模块的修改：用户认证、订单状态、消息通知。

她跑来找老李求助："李总，三个模块要改，时间不够怎么办？"

老李问："你打算怎么用 AI？"

小想："开一个 Chat，把三个模块的文件都挂进去，让 AI 一次性改完。"

老李摇头："这又是贪多嚼不烂。你改三个功能，开三个 Chat，每个 Chat 只挂那个功能相关的文件。"

小想困惑："开三个 Chat？"

老李解释："你想想，你查三个案例的资料，是开一个浏览器窗口来回切换，还是开三个窗口并列对比？肯定是后者。AI Chat 也一样，隔离任务，上下文干净，输出质量高。"

小想尝试了：开三个 Chat，分别挂载 auth 目录、order 目录、message 目录。每个 Chat 只处理那个模块的修改。

半小时后，小想回来报告："成功了！每个 Chat 的输出都很精准，没有混淆。我还对比了三个 Chat 的修改风格，做了统一调整。"

老李点头："这就是多开 Chat 平行探索。每个 Chat 做一件事，最后择优合并。比一个大杂烩 Chat 质量高得多。"

### 故事要点

| 要点 | 经验 |
|------|------|
| 多任务隔离 | 三个功能开三个 Chat |
| 平行探索 | 每个 Chat 只挂相关文件 |
| 择优合并 | 输出对比后统一风格 |
| 类比强化 | 多窗口查资料对比 |

### 对话金句

小想："三个模块要改，时间不够怎么办？"

老李："开三个 Chat，每个 Chat 只挂那个功能相关的文件。"

小想："每个 Chat 做一件事，最后择优合并。"

## 故事四：新人入职的信息喂养教训

### 场景设定

- **人物**：老李回忆、新入职的产品助理小王
- **背景**：小王入职第一天被塞了公司五年所有文档
- **时间**：两年前的回忆

### 故事正文

老李在茶水间回忆往事："两年前，有个新入职的产品助理叫小王。他第一天来，我们把公司五年的所有文档、会议纪要、PRD 存档都塞给他了。"

小想问："让他快速了解公司？"

老李摇头："结果他崩溃了。第一周啥都干不了，整天说'信息量太大消化不了'。第二周他离职了。"

小想惊讶："离职了？"

老李说："信息焦虑。像喂饭一样，第一天塞十碗饭，谁不吐？好经理带新人，第一天给架构图，第二天给当前项目文档，第三天才看历史存档。按节奏喂，不一次塞完。"

小想领悟："AI 也一样……"

老李点头："AI 就像新员工。你一次性塞整个仓库历史，它也会'幻觉'。分阶段喂，当前任务先给，历史必要时才查。"

### 故事要点

| 要点 | 教训 |
|------|------|
| 信息焦虑 | 新人被信息淹没会崩溃 |
| 分阶段喂养 | 第一天架构，第二天当前项目，第三天历史 |
| AI 同理 | AI 被上下文过载也会"幻觉" |
| 节奏意识 | 按任务节奏喂，不一次塞完 |

### 对话金句

老李："第一天塞十碗饭，谁不吐？好经理带新人，按节奏喂，不一次塞完。"

小想："AI 也一样……"

老李："AI 就像新员工。你一次性塞整个仓库历史，它也会'幻觉'。"
