# 故事场景：老板说"做个小系统应该很便宜"

> 素材类型：故事场景
> 知识点：A9-01 系统开门成本
> 更新日期：2026-04-19

---

## 场景描述

老板想做一个简单的内部管理系统，问 IT 团队："做个小系统应该很便宜吧？"IT 报价远超老板预期，引发误解冲突。

---

## 对话脚本

**老板**：我们想做一个简单的内部审批系统，就几页表单、几个按钮那种。做个小系统应该很便宜吧？预算两三万够不够？

**IT 经理**：老板，这个预算可能不够。光是基础设施就要不少钱。

**老板**：基础设施？不就是一台服务器吗？现在云服务器很便宜啊。

**IT 经理**：服务器确实不贵，但不止一台服务器。给您算算：

| 项目 | 说明 | 一年费用 |
|------|------|---------|
| **云服务器** | 最基础的 ECS，2核4G | ¥3,000+ |
| **域名** | 内部系统可能要用域名 | ¥500+ |
| **SSL 证书** | HTTPS 安全访问，企业级 OV 证书 | ¥1,000+ |
| **数据库** | RDS 数据库服务 | ¥2,000+ |
| **对象存储** | 存附件、文件 | ¥500+ |
| **监控告警** | Prometheus + Grafana 或云监控 | ¥1,000+ |
| **日志系统** | ELK Stack 或云日志服务 | ¥1,000+ |
| **数据备份** | 自动备份存储空间 | ¥500+ |
| **CDN**（可选） | 如果有外部访问加速需求 | ¥500+ |

**老板**：加起来要一万多？这还没写一行代码呢？

**IT 经理**：是的。这叫"开门成本"——还没开门营业就要花的钱。

---

## 核心洞察

### 开门成本的构成

| 类别 | 具体项目 | 为什么必须有 |
|------|----------|-------------|
| **计算资源** | 服务器、容器服务 | 运行代码的基础 |
| **数据存储** | 数据库、对象存储 | 存业务数据 |
| **网络基础设施** | 域名、DNS、SSL证书 | 让用户能安全访问 |
| **可观测性** | 监控、日志、告警 | 出问题能排查 |
| **数据安全** | 备份、灾备方案 | 数据不能丢 |
| **安全合规** | 身份认证、访问控制 | 不能被随便访问 |

### 常见误解纠正

| 误解 | 正确理解 |
|------|----------|
| "服务器就是基础设施的全部" | 域名、证书、监控、日志、备份都是基础设施 |
| "小系统成本低" | 开门成本是相对固定的，小系统也要这些基础 |
| "云服务便宜" | 云服务单价便宜，但项目多了总价不便宜 |
| "免费的够用" | 免费 SSL 证书（如 Let's Encrypt）适合个人站，企业系统建议用 OV/EV 证书 |
