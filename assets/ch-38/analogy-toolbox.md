# 类比：设计系统 = 统一工具箱

> 知识点：B6-05 — 设计系统
> 类比类型：结构解释型类比
> 适用章节：第 3 节（核心概念）、第 4 节（误解纠正）

---

## 类比核心

**设计系统就像团队统一的工具箱。**

不是每个人各自准备工具，而是有一个标准化的工具箱，所有人用同样的工具，按同样的方式存放。

---

## 类比展开

### 有无设计系统的对比

**没有设计系统（各自准备工具）：**

```
设计师 A 的工具箱：
  → 按钮：圆角 4px、蓝色 #1E90FF
  → 卡片：阴影 2px、灰色边框
  → 字体：思源黑体 14px

设计师 B 的工具箱：
  → 按钮：圆角 8px、绿色 #00AA55
  → 卡片：阴影 4px、无边框
  → 字体：苹方 16px

结果：
  → 同一个产品，两种风格
  → 开发问「用哪个？」
  → 用户困惑「这还是同一个 App 吗？」
```

**有设计系统（统一工具箱）：**

```
团队统一工具箱：
  → 按钮：圆角 6px、品牌蓝 #0066CC（标准件）
  → 卡片：阴影 2px、浅灰边框（标准件）
  → 字体：系统字体 14px（标准件）
  → 使用指南：按钮用于主操作（说明书）

结果：
  → 所有人用同样的工具
  → 开发不需要问「用哪个」
  → 用户看到一致的产品风格
```

### UI Kit 与设计系统的区别类比

| UI Kit | 设计系统 |
|--------|---------|
| **只有工具**（零件盒） | **工具 + 说明书 + 使用规范**（完整工具箱系统） |
| 「这里有螺丝」 | 「这是 M4 螺丝，用于固定面板，拧紧力矩 2N」 |
| 设计师自己决定怎么用 | 有明确的使用指南 |

```
类比：
  UI Kit      = 只给你一堆零件
  设计系统    = 给你零件 + 说明书 + 使用规范 + 维护手册
```

## 类比价值

### 为什么用这个类比？

1. **PM 理解工具协作**：工具箱是日常协作场景
2. **直观展示「统一」的价值**：避免各自准备工具的混乱
3. **区分 UI Kit 与设计系统**：零件盒 vs 完整工具箱系统

### 类比边界

| 适用 | 不适用 |
|------|--------|
| 解释什么是设计系统 | 解释复利效应 |
| 解释 UI Kit 区别 | 解释具体组件设计 |
| 展示协作价值 | 展示 ROI 数据 |

## 可能的质疑

### 质疑：工具箱是静态的，设计系统要持续维护

**回应**：类比侧重「统一存放、统一使用」的核心逻辑。设计系统确实需要持续维护（就像工具箱要定期补充、更新），但类比的核心价值不变。
