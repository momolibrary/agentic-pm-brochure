## 类比内容

**系统性发现异常和边界遗漏，就像飞行员起飞前的检查清单。**

### 场景展开

飞行员起飞前不靠"感觉"，而是逐项检查：

```
□ 发动机状态□ 油量充足□ 仪表正常□ 天气条件□ 紧急预案...
```

产品经理上线前也应该有"检查清单"：

```
□ 网络断开时显示什么□ 超时时如何提示□ 权限不足时如何处理□ 数据不存在时如何响应□ 空值输入如何校验□ 最大值输入如何限制□ 并发操作如何处理□ 超大数据量如何分页□ 跨时区时间如何计算□ 金额计算精度如何保证
```

### 对比解读

**为什么飞行员不靠感觉？**
- 漏一项就可能出人命
- 检查清单是"生命的保障"

**为什么产品经理也需要清单？**
- 漏一个异常场景就可能导致用户投诉、数据丢失、业务中断
- 清单是"系统稳定性的保障"

### 清单分类

**异常清单（网络/系统层面）**：

| 异常类型 | 检查点 |
|---------|-------|
| 网络断开 | 页面空白？有提示？有重试？ |
| 超时 | 卡住？报错？自动重试？ |
| 权限不足 | 拒绝访问？提示升级权限？ |
| 数据不存在 | 报404？返回空数据？显示默认值？ |
| 第三方接口失败 | 系统崩溃？降级处理？ |

**边界清单（数据层面）**：

| 边界类型 | 检查点 |
|---------|-------|
| 空值 | 是否校验？如何提示？ |
| 最大值 | 是否限制？如何截断？ |
| 并发 | 是否加锁？如何提示冲突？ |
| 大数据量 | 是否分页？是否异步处理？ |
| 时区 | 是否统一？如何转换？ |
| 精度 | 是否用定点数？如何舍入？ |

### AI 辅助生成

现代飞行员有"飞行助手"提示异常风险。
产品经理可以用 AI 辅助生成异常和边界场景：

```
输入：需求描述 + 正常流程图输出：AI生成的异常场景清单 + 边界条件清单
```
