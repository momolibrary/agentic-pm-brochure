# 类比：边界条件遗漏 = 驾驶只考晴天路况

> 素材类型：类比
> 知识点：C3-04 边界陷阱
> 适用章节：CH-50 Section 2

---

## 类比内容

**忽略边界条件的产品设计，就像考驾照只考晴天路况，不考雨天、夜间、拥堵。**

### 场景展开

产品经理设计功能，只考虑"正常场景"，就像驾校只教晴天：

| 驾校只教晴天 | 驾校完整培训 |
|-------------|-------------|
| 平直道路行驶 | 雨天湿滑路面 |
| 白天视线清晰 | 夜间灯光使用 |
| 正常速度驾驶 | 拥堵时低速行驶 |
| 单车道行驶 | 多车道变道 |
| 无行人干扰 | 突发行人横穿 |

对应到产品边界条件：

| 产品只考虑正常 | 产品完整考虑边界 |
|---------------|-----------------|
| 单人正常操作 | 两人同时修改（并发） |
| 几十条数据查询 | 10万条数据导出（大数据量） |
| 正常长度输入 | 超长字符串、特殊字符（极端输入） |
| 同一时间操作 | 跨时区协作（时区问题） |
| 整数金额计算 | 浮点数金额（精度问题） |

### 核心教训

> 驾校不教雨天，新手遇到雨天就出事故。产品不考虑边界，上线遇到边界就崩溃。

**五大边界陷阱**：

| 边界类型 | 典型问题 | 后果 |
|---------|---------|------|
| **并发问题** | 两人同时修改同一订单 | 数据覆盖/丢失 |
| **大数据量** | 导出10万条数据 | 内存溢出/超时 |
| **极端输入** | 用户输入10000字评论 | 存储失败/截断 |
| **时区问题** | 海外用户下单时间显示错误 | 业务时间计算错误 |
| **精度问题** | 金额用浮点数计算 | 0.1+0.2≠0.3 |

### 人物对话

> "小想，你设计的这个功能挺好，但就像驾校只教晴天开车。雨天怎么办？夜间怎么办？拥堵怎么办？边界条件不考虑，上线就出事故。"

小想：
> "边界条件？哪些算边界？"

老李：
> "并发、大数据量、极端输入、时区、精度——这五个，我叫它们'边界五虎'。"
