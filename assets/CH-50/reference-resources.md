### 方法框架

**异常处理 Checklit 模板**：

```markdown
# 异常处理检查清单

## 网络层面
- [ ] 网络断开时：页面是否有提示？是否有重试按钮？
- [ ] 超时时：是否有加载动画？是否自动重试？
- [ ] 第三方接口失败时：是否有降级方案？是否有错误上报？

## 权限层面
- [ ] 权限不足时：是否有友好提示？是否引导解决？
- [ ] 未登录时：是否跳转登录？是否保存当前操作？

## 数据层面
- [ ] 数据不存在时：是否有默认展示？是否有引导提示？
- [ ] 数据格式错误时：是否有具体提示？是否有示例？
```

**边界条件 Checklist 模板**：

```markdown
# 边界条件检查清单

## 输入边界
- [ ] 空值：是否校验？如何提示？
- [ ] 最大值：是否限制？如何截断？
- [ ] 特殊字符：是否过滤？是否提示？

## 操作边界
- [ ] 并发操作：是否加锁？是否提示冲突？
- [ ] 大数据量：是否分页？是否异步处理？

## 时间边界
- [ ] 时区问题：是否统一存储？是否本地显示？
- [ ] 跨年/跨月：是否正确计算？

## 数值边界
- [ ] 精度问题：金额是否用定点数？是否正确舍入？
- [ ] 数值溢出：是否检查范围？
```

## 边界条件相关资源

### 并发问题

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| 乐观锁 | 低冲突场景 | 版本号/时间戳检查，冲突时提示 |
| 悲观锁 | 高冲突场景 | 加锁编辑，防止同时操作 |
| 最后写入优先 | 简单场景 | 后保存覆盖前保存，可能数据丢失 |
| 合并方案 | 协作场景 | 显示冲突，用户手动选择 |

### 大数据量问题

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| 分页查询 | 列表展示 | 每次只加载部分数据 |
| 异步处理 | 导出/导入 | 后台处理，进度提示 |
| 流式处理 | 文件生成 | 逐行生成，不一次性加载 |
| 缓存预热 | 高频查询 | 预加载热点数据 |

### 精度问题

| 问题 | 说明 | 解决方案 |
|------|------|---------|
| 0.1 + 0.2 ≠ 0.3 | 浮点数精度误差 | 使用 BigDecimal |
| 金额舍入错误 | 四舍五入导致误差 | 统一舍入规则（如银行家舍入法） |
| 时间精度丢失 | 毫秒/秒级精度不一致 | 统一时间精度单位 |

## 行业最佳实践

### 异常处理原则

1. **用户友好**：异常信息不暴露技术细节，用通俗语言描述
2. **可恢复性**：异常发生后，用户可以重试或恢复操作
3. **降级处理**：关键服务失败时，有备选方案
4. **上报机制**：异常发生后，系统自动上报，便于排查

### 边界条件原则

1. **前端校验**：输入边界在前端实时校验，减少后端压力
2. **后端兜底**：前端校验失败或被绕过时，后端二次校验
3. **设计时考虑**：边界条件在设计阶段就识别，不是测试阶段发现
4. **自动化检测**：用工具/脚本自动化检测边界遗漏
