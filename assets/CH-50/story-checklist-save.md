# 故事：用检查清单拯救上线

> 素材类型：故事/案例
> 知识点：C3-03 + C3-04 系统性发现方法
> 适用章节：CH-50 Section 3

---

## 故事内容

### 背景

小想准备上线一个新的"用户管理系统"。上线前，老李让他做一次"异常和边界检查"。

小想：
> "检查什么？我觉得流程都设计好了。"

老李：
> "上次上线即崩溃，就是因为你靠'感觉'检查。这次用清单。"

### 检查清单方法

老李给小想两张清单：

**异常清单（网络/系统层面）**：

```
□ 网络断开时页面是否有提示？
□ 超时时是否有加载动画？
□ 权限不足时是否有友好提示？
□ 数据不存在时是否有默认展示？
□ 第三方接口失败时是否有降级方案？
```

**边界清单（数据层面）**：

```
□ 空值输入是否校验？
□ 最大值输入是否限制？
□ 并发操作是否处理？
□ 大数据量是否分页？
□ 时区是否统一？
□ 精度是否用定点数？
```

### AI辅助生成

小想把需求描述发给AI：
> "我有一个用户管理系统，包含登录、查询用户列表、编辑用户信息、批量导入用户。请帮我生成异常场景清单和边界条件清单。"

AI返回：

**异常场景清单**：

| 场景 | 异常类型 | 建议处理 |
|------|---------|---------|
| 登录时网络断开 | 网络异常 | 提示"网络异常，请检查网络" |
| 登录时服务器超时 | 超时 | 显示"登录中..." + 自动重试 |
| 登录时密码错误 | 业务异常 | 提示"密码错误，请重试" |
| 查询用户时数据库连接失败 | 系统异常 | 提示"系统异常，请稍后再试" |
| 编辑用户时权限不足 | 权限异常 | 提示"您没有编辑权限" |
| 批量导入时文件格式错误 | 数据异常 | 提示"文件格式不正确，请使用CSV格式" |

**边界条件清单**：

| 场景 | 边界类型 | 建议处理 |
|------|---------|---------|
| 用户名超长（100+字符） | 极端输入 | 前端限制50字符 + 提示剩余字数 |
| 用户名包含特殊字符 | 极端输入 | 校验规则：只允许字母、数字、下划线 |
| 查询用户列表无结果 | 空值 | 显示"暂无用户数据" + 添加引导 |
| 两人同时编辑同一用户 | 并发 | 乐观锁 + 冲突提示 |
| 批量导入10000条用户 | 大数据量 | 异步处理 + 进度提示 + 邮件通知 |
| 海外用户查看创建时间 | 时区 | 后端存UTC + 前端按用户时区显示 |

### 发现遗漏

小想照着清单逐项检查，发现遗漏了12处：

| 遗漏项 | 原设计 | 补充设计 |
|-------|-------|---------|
| 网络断开提示 | 页面空白 | 提示"网络异常" + 重试按钮 |
| 登录超时处理 | 页面卡住 | 显示加载动画 + 自动重试 |
| 密码错误提示 | 显示"Error" | 提示"密码错误，剩余X次" |
| 查询无结果 | 空列表 | 显示"暂无数据" + 添加引导 |
| 用户名长度限制 | 无限制 | 前端限制50字符 |
| 特殊字符校验 | 无校验 | 前端实时校验 + 错误提示 |
| 并发编辑冲突 | 无处理 | 乐观锁 + 冲突提示 |
| 批量导入进度 | 无提示 | 进度条 + 完成邮件通知 |

### 上线结果

上线后一周，客服投诉量比上次减少90%。运维反馈：
> "这次上线很稳，没什么异常投诉。"

老李：
> "这就是检查清单的作用。你不再靠感觉，而是靠系统。"

### 核心教训

1. **靠感觉检查 = 遗漏**：人脑无法穷举异常和边界场景
2. **清单是系统性工具**：异常清单 + 边界清单 + AI辅助生成
3. **提前发现 = 减少投诉**：上线前发现遗漏，比上线后修复成本更低
