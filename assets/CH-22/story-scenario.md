## 主场景：小想看到开发屏幕

### 场景描述

小想第一次走进开发团队的办公室。

路过一个开发的工位，看到屏幕上花花绿绿的界面——左边是一排排彩色的代码，右边有黑底白字的窗口，底部还有各种小图标和数字跳动。

小想好奇地问：「这是什么软件？看起来挺酷的。」

开发笑了笑：「这是我的 IDE，写代码的地方。」

小想：「IDE？」

开发：「集成开发环境。你看，左边是我写的代码，颜色不一样是因为不同类型的代码高亮显示；右边那个黑窗口是终端，我可以直接跑命令；下面这个是 Git 状态，能看到我改了哪些文件。」

小想：「哦，原来不是记事本啊。」

开发：「记事本只能写字。这东西能写字、能调试、能运行、还能帮我管版本——是开发的「工作台」。」

## 场景变体：产品经理装 Cursor

### 场景描述

老李让小想去体验一下开发工具。

小想：「我不会写代码啊，装个 IDE 有什么用？」

老李：「你不用写代码，但你可以用 AI。现在有种叫 Cursor 的 IDE，内置了 AI。你可以在里面让 AI 帮你写需求文档、画原型代码，甚至验证一些想法。」

小想装了 Cursor，打开侧边栏的 AI 聊天窗口。

输入：「帮我写一个登录页面的需求文档。」

AI 在编辑器里直接生成了一份结构化的需求文档，格式还是团队常用的模板。

小想：「原来 IDE 不只是开发用的……」

老李：「对。AI 时代的 IDE，产品经理也能用。你不用会写代码，但你可以让 AI 帮你做事情。」

## 场景变体：误解纠正

### 场景描述

小想和一个开发对接需求。

小想：「你说的那个改接口，是不是在你们那个写代码的软件里改一下就行？」

开发：「不是『改一下』——我要先在 IDE 里找到那段代码，理解上下文，改完之后在 IDE 里调试确保没问题，再用 IDE 的 Git 功能提交……」

小想：「啊，原来这么多步骤。」

开发：「这就是为什么 IDE 叫『集成开发环境』——它把整个开发流程的工具都整合在一起了。」

---

## 场景要点提炼

1. **视觉冲击**：IDE 界面对非技术人员来说很「酷」，这是好奇心入口
2. **误解触发**：小想的「记事本」误解是常见认知偏差，需要纠正
3. **AI 转折**：Cursor 的引入让产品经理也能用 IDE，拓宽认知边界
4. **流程揭示**：开发解释 IDE 的多步骤功能，让 PM 理解「集成」的含义
