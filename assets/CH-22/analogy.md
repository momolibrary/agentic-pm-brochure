## 主类比：IDE vs Figma

### 核心类比

**IDE 是开发者的「工作台」，就像 Figma 是设计师的「工作台」。**

| Figma（设计师） | IDE（开发者） |
|----------------|--------------|
| 编辑设计稿 | 编辑代码 |
| 组件库 | 代码库/模板 |
| 协作评论 | Git 提交记录 |
| 版本历史 | Git 分支/历史 |
| 插件市场 | IDE 插件市场 |
| 输出切图/样式代码 | 输出编译产物 |

### 为什么这个类比好？

1. **产品经理熟悉 Figma**：PM 看设计稿、提反馈都在 Figma，知道它的角色
2. **对应关系清晰**：两边都有「编辑」「协作」「版本」「插件」「输出」
3. **功能定位相近**：都是专业人士的核心工作台，不是「随手用的工具」

### 类比话术示例

> 「你每天在 Figma 看设计稿、写评论，设计师在 Figma 做设计——Figma 是他们的工作台。开发也有自己的工作台，叫 IDE。他们写代码、调试、看 Git 历史都在这里面。你不用学会用它，但要知道它是什么，就像你知道 Figma 是什么一样。」

## 次类比：IDE vs 剪辑软件

### 类比思路

**IDE 之于开发，就像剪辑软件之于视频剪辑师。**

| 剪辑软件（Premiere/剪映） | IDE |
|-------------------------|-----|
| 时间线编辑窗口 | 代码编辑窗口 |
| 预览窗口 | 运行结果/浏览器 |
| 特效面板 | 插件/工具面板 |
| 渲染输出 | 编译输出 |
| 项目素材库 | 文件树/代码库 |

### 适用场景

如果读者是视频背景的产品经理，可以用这个类比。

## 次类比：IDE vs 办公软件套件

### 类比思路

**IDE 是「开发的 Office」——把所有需要的工具整合到一个窗口里。**

以前没有 Office，你要写文档用 Word，算表格用 Excel，做演示用 PowerPoint——三个软件来回切。

有了 Office 之后，你可以在一个界面里切换文档、表格、演示，数据还能互通。

IDE 也是这个思路：
- 写代码（编辑器）
- 调程序（调试器）
- 管版本（Git）
- 跑命令（终端）

全在一个窗口，不用来回切软件。

### 适用场景

适合非技术背景的产品经理，用熟悉的办公场景类比。

## 反类比：IDE 不是「记事本」

### 常见误解

> 「IDE 就是写代码的记事本吧？」

### 正确理解

记事本只能「写字」，IDE 是「写字 + 调试 + 运行 + 管理 + AI 辅助」的综合台。

类比：
- 记事本 = 只能写字的白纸
- IDE = 有格子、有公式栏、有图表工具、能自动计算、能保存版本的 Excel

**不是「写」，是「写 + 做 + 管」。**
