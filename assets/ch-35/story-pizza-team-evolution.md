# 故事场景：披萨团队演变

> 知识点：B5-07
> 用途：第 4 节误解案例：「人越多效率越高」误解纠正

---

## 场景背景

**时间**：某公司的产品团队会议
**人物**：老李（产品总监）、小想（实习生）、CTO

---

## 故事内容

### 开场对话

会议室里，产品团队正在讨论人力需求。

小想发言：「我觉得我们要加速产品迭代，应该多招几个产品经理和开发，把团队扩大到 20 人。」

老李打断：「你确定 20 人比现在的 8 人效率更高？」

小想理所当然：「人多力量大嘛，肯定更快。」

老李笑了笑：「你知道 Amazon 的「披萨团队」规则吗？」

小想：「听说过，就是团队规模不超过两张披萨能喂饱的人数。」

「为什么要这样？」老李追问。

小想想了想：「可能是为了沟通效率？」

老李点头：「不只是沟通效率。让我给你们算一笔账。」

---

## 沟通成本定律

### 公式解释

老李在白板上写下：

```
沟通通道数 = n(n-1)/2
```

「这是什么？」小想问。

「这是团队内潜在的沟通通道数量，n 是团队人数。」

老李开始计算：

| 团队规模 | 沟通通道数 | 说明 |
|---------|-----------|------|
| 2 人 | 1 | 一条线 |
| 3 人 | 3 | 三角形 |
| 5 人 | 10 | 五边形 + 交叉线 |
| 8 人（当前） | 28 | 已经有点复杂 |
| 10 人 | 45 | 沟通开始混乱 |
| 20 人（小想提议） | 190 | **沟通灾难** |

小想看到 190 这个数字，愣住了。

「所以你想让团队从 28 条沟通通道，变成 190 条？」老李问。

### Brooks 定律

CTO 插话：「你们还记得 Brooks 定律吗？」

小想：「什么定律？」

CTO 解释：「**向延迟的项目添加人力，只会让它更延迟。** 这句话来自 Fred Brooks 的《人月神话》。」

「为什么？」小想不解。

「两个原因：
1. 新人需要时间融入，这段时间老成员要花时间教他们，效率下降
2. 沟通成本随人数平方增长，大家花更多时间沟通，更少时间干活」

---

## 披萨团队的未来演变

老李继续：「所以 Amazon 的披萨团队规则不是随便定的。两张披萨 ≈ 6-8 人，沟通通道控制在 15-28 条以内。」

小想：「但披萨团队是十几年前的概念了，现在 AI Agent 出现，情况会不会变化？」

老李点头：「正是。披萨团队会演变成什么？」

### 演变路径

老李画了一个演变图：

```
传统披萨团队（2010年代）
┌─────────────────────────────────┐
│  6-8 人团队，全靠 Human 沟通    │
│  沟通通道：15-28 条              │
│  每个人都有自己的技能边界         │
└─────────────────────────────────┘
            ↓
Human+Agent 披萨小组（2020年代中）
┌─────────────────────────────────┐
│  2-3 人 + 每个 Human 配 Agent   │
│  Human 间沟通通道：3 条           │
│  Human-Agent 沟通：内部高效      │
│  覆盖能力 ≈ 传统 6-8 人团队      │
└─────────────────────────────────┘
            ↓
超级个体小组（2030年代）
┌─────────────────────────────────┐
│  1-2 人 + 各自 Agent 团队       │
│  Human 间沟通通道：1 条           │
│  每个 Human 可覆盖整个业务域     │
│  产出 ≈ 传统 10-20 人团队        │
└─────────────────────────────────┘
```

### 关键洞察

小想恍然大悟：「所以披萨团队的规则不是消失，而是升级了！」

老李：「对。核心不变——**控制沟通复杂度**。但手段变了——用 Agent 替代 Human，减少「肉脑袋间」的沟通。」

---

## 反例：为什么不能直接砍人？

小想追问：「那我们现在把团队砍到 2 人，用 Agent 补上？」

CTO 摇头：「还不行。有两个前提条件。」

**前提一：判断力足够**
- 2 个人能否做出 8 个人的判断？
- 如果判断力不够，Agent 只会执行垃圾指令

**前提二：Agent 编排成熟**
- 你会配置 Agent 吗？知道任务边界吗？
- 这是新技能，大多数人还不熟练

老李补充：「所以披萨团队的演变是渐进的。先从 8 人 → 3 人 + Agent，验证可行后再继续减。不是一刀切。」

---

## 复盘总结

小想总结：「所以误解是——以为团队人越多效率越高。正确理解是——沟通成本与人数平方正相关，Human+Agent 模式可以在不增加沟通负担的前提下放大产出。」

老李点头：「记住这个公式：**沟通通道 = n(n-1)/2**。每次想加人，先算一下沟通成本。」
