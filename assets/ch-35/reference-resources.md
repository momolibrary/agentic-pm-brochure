## Agent 编排框架

### 框架对比（2025）

| 框架 | 适用场景 | 特点 |
|------|---------|------|
| CrewAI | 角色型 Agent 团队，快速原型 | 简单易学，适合内容创作、客服 |
| LangGraph | 复杂状态流，生产级系统 | 状态管理强，适合审批流程、多步推理 |
| AutoGen | 会话型 Agent，研究实验 | 会话模式强，适合代码审核、协作推理 |

### 选择建议

- 快速开始 → CrewAI
- 生产级复杂流 → LangGraph
- 研究实验 → AutoGen

## 相关概念

### Dunbar's Number

- 人类能维护的稳定社会关系约 150 人
- 小团队（<10 人）沟通成本可控
- 大团队需要层级结构降低沟通复杂度

### 披萨团队演变

| 时代 | 团队形态 |
|------|---------|
| 2010 年代 | 6-8 人披萨团队，全靠 Human 沟通 |
| 2020 年代中 | 2-3 人 + Agent，Human-Agent 内部高效 |
| 2030 年代 | 1-2 人 + Agent 团队，超级个体覆盖业务域 |

## 误解纠正来源

### 误解一：「AI 替代人」

| 来源 | 纠正观点 |
|------|---------|
| Human judgment + AI execution 研究 | 不是替代，是放大 |
| 核心洞察 | Human 判断力稀缺，Agent 执行力放大 |

### 误解二：「团队人越多效率越高」

| 来源 | 纠正观点 |
|------|---------|
| Brooks 定律 | 添加人力会让延迟项目更延迟 |
| 沟通成本公式 | n(n-1)/2，沟通成本随人数平方增长 |
| 披萨团队规则 | 控制团队规模，控制沟通复杂度 |
