# 避免 AI 幻觉对需求文档的污染

## 概述

AI 幻觉是 LLM 的固有缺陷，产品经理需要建立防范机制，避免幻觉内容进入正式需求文档。

---

## 策略一：所有关键信息标注来源

### 原理

任何进入需求文档的「关键信息」，都必须有可追溯的来源。

### 什么是「关键信息」

| 信息类型 | 说明 |
|----------|------|
| 统计数据 | 用户数量、市场份额、转化率等 |
| 用户引用 | 用户反馈、用户语录等 |
| 外部引用 | 竞品信息、行业标准、法规条文等 |
| 时间节点 | 发布日期、版本号、里程碑时间等 |

### 标注格式示例

```markdown
# 需求文档片段

用户留存数据显示，新用户 7 日留存率为 42%。
> 来源：数据分析平台 2026-04-15 导出报告（附件：retention-report.xlsx）

竞品 A 在注册流程中采用「邮箱验证」方式。
> 来源：竞品 A 官网注册页面截图（附件：competitor-a-register.png）
> 验证日期：2026-04-20

根据《个人信息保护法》第 XX 条，需要告知用户数据处理方式。
> 来源：《个人信息保护法》全文（https://xxx.gov.cn/law/xxx）
> 验证日期：2026-04-18
```

### 执行规范

1. **无来源标注的信息，视为「未验证」**
2. **标注来源后，其他阅读者可以独立验证**
3. **来源标注应包含：来源名称、获取时间、可追溯路径（链接/附件）**

## 策略二：AI 生成内容标记为「待验证」

### 原理

所有 AI 生成的「关键信息」，在验证前应标记为「待验证」状态，提醒后续处理者。

### 标记方式

```markdown
# 需求文档片段（AI 辅助草稿）

## 用户画像分析

核心用户群体为 25-35 岁职场白领，占比约 60%。
[⚠️ 待验证] 数据来源未确认，需回源验证

主要使用场景为通勤时段（早 7-9 点，晚 5-7 点）。
[⚠️ 待验证] 需通过用户调研或数据分析验证

## 竞品对比

竞品 X 的注册流程约需 3 分钟完成。
[⚠️ 待验证] 需实测验证，截图留档
```

### 执行规范

1. **AI 输出的「关键信息」，统一标记 `[⚠️ 待验证]`**
2. **验证后，更新标记为 `[✅ 已验证] 来源: ...`**
3. **无法验证的信息，标记为 `[❌ 不可用] 原因: ...`**

## 策略三：建立验证清单

### 原理

为需求文档建立标准化的「验证清单」，在文档发布前逐项检查。

### 验证清单模板

```markdown
## 需求文档验证清单

### 信息来源验证

- [ ] 所有统计数据已标注来源
- [ ] 所有用户引用已标注来源（用户 ID/访谈记录）
- [ ] 所有外部引用已标注来源（链接/文档）
- [ ] 所有时间节点已标注来源（日历/公告）

### 内容验证

- [ ] AI 生成的关键信息已标记「待验证」或「已验证」
- [ ] 无法验证的信息已标记「不可用」并移除或备注
- [ ] 专业领域内容已由专家确认

### 格式验证

- [ ] 来源标注格式统一
- [ ] 验证状态标记清晰可见
- [ ] 附件已上传并链接正确

### 最终确认

- [ ] 所有「待验证」项已处理完成
- [ ] 文档审核人已确认验证完整性
```

### 执行规范

1. **文档初稿完成后，必须执行验证清单检查**
2. **所有「待验证」项处理完成后，才能进入正式评审**
3. **验证清单随文档存档，便于后续追溯**

## 策略四：关键决策不能只依赖 AI 输出

### 原理

影响产品方向、资源分配、用户体验的关键决策，必须有「非 AI」的信息输入。

### 关键决策类型

| 决策类型 | 说明 | 需要的非 AI 信息 |
|----------|------|------------------|
| 功能优先级 | 决定开发什么功能 | 用户调研、数据分析、业务目标 |
| 用户取舍 | 决定服务哪类用户 | 用户画像数据、市场分析 |
| 技术方案 | 决定采用什么技术架构 | 技术评估、性能测试、专家意见 |
| 时间节点 | 决定什么时候上线 | 项目排期、资源情况、风险评估 |

### 决策输入规范

```
关键决策输入来源必须满足：
  - 至少 2 个独立信息来源
  - 至少 1 个「非 AI」来源（调研、数据、专家）
  - 所有来源已验证可信

决策记录必须包含：
  - 决策依据的信息来源列表
  - 信息来源的验证状态
  - 决策过程的参与人员
```

### 执行规范

1. **关键决策前，明确「决策依据」的信息来源**
2. **确保至少有 1 个「非 AI」验证来源**
3. **决策记录中保留验证过程，便于后续复盘**

## 幻觉防范流程图

```
AI 输出内容
    ↓
判断内容类型
    ↓
├─ 关键信息 → 标记「待验证」 → 验证 → 标记来源
│                                  ↓
│                           ├─ 验证成功 → 进入文档
│                           └─ 验证失败 → 移除或备注
│
├─ 一般描述 → 基于常识判断 → 可用或调整
│
└─ 专业内容 → 专家确认 → 进入文档（附专家意见）
```
