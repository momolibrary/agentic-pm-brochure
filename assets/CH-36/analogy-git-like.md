## 类比一：文档修订留痕

### 核心类比

**Git 版本管理 = 文档修订留痕**

| Git 概念 | 文档类比 |
|---------|---------|
| **版本库** | 文档的完整修订历史档案 |
| **Commit** | 每次保存文档的修改记录 |
| **分支** | 不同作者同时修改文档的副本 |
| **Pull Request** | 编辑审核修改建议 |
| **Merge** | 将多位作者的修改合并到主文档 |
| **Diff** | 文档修订痕迹对比（红色删除、绿色新增） |
| **冲突** | 两人同时修改同一段落 |
| **Rollback** | 发现问题，撤销修改 |

### 类比说明

需求变更就像文档修订：
- 每次改动要有记录（像 Word 的修订痕迹）
- 改动原因要说明（像修订说明）
- 改动内容可追溯（像版本历史）
- 改动需要审核（像编辑审核修改）

## 类比二：装修施工记录

### 核心类比

**Git 版本管理 = 装修施工记录**

| Git 概念 | 装修类比 |
|---------|---------|
| **项目** | 一套房子装修工程 |
| **源代码** | 设计师的图纸 |
| **Commit** | 每天的施工日志记录 |
| **分支** | 不同装修方案（现代风格分支、中式风格分支） |
| **Merge** | 将多个房间的装修方案整合 |
| **Rollback** | 发现问题，拆除重装 |
| **Push** | 向业主提交施工进度报告 |
| **Pull** | 获取业主的新要求 |
| **Issue** | 施工问题单（发现墙面裂缝、需要整改） |

### 类比说明

需求变更就像装修变更：
- 每次改动要记录施工日志（像 commit）
- 改动原因要说明（为什么要改）
- 改动内容要对比（原图纸 vs 新图纸）
- 改动需要审核（业主确认）

## 类比三：餐厅菜单管理

### 核心类比

**Git 版本管理 = 餐厅菜单管理**

| Git 概念 | 餐厅类比 |
|---------|---------|
| **版本库** | 菜单的历史版本档案 |
| **Commit** | 每次菜单更新的记录 |
| **分支** | 不同季节菜单（夏季菜单、冬季菜单） |
| **Pull Request** | 新菜品试做、厨师长审核 |
| **Merge** | 将新菜品加入正式菜单 |
| **Diff** | 菜单对比（新增菜品、删除菜品） |
| **Issue** | 顾客反馈单（菜品口味问题） |

### 类比说明

需求变更就像菜单更新：
- 每次菜品更新要记录（像 commit）
- 更新原因要说明（为什么加这道菜）
- 更新内容要对比（菜单变化）
- 更新需要审核（厨师长确认）

## Git-like 需求管理核心概念

### Issue = 需求变更请求

类比装修施工的"施工问题单"：

**Issue 模板结构**：
- 变更标题（问题单标题）
- 变更原因（问题描述）
- 变更来源（问题来源）
- 影响范围（影响哪些房间）
- 变更内容（整改方案）

### PR = 变更评审

类比装修施工的"整改方案审核"：

**PR 评审清单**：
- 变更原因是否合理（问题是否真实）
- 影响范围是否评估（涉及哪些房间）
- 技术可行性是否确认（施工队能否实施）
- 排期是否协调（施工时间安排）

### Commit Log = 变更记录

类比装修施工的"施工日志"：

**Commit Log 格式**：
- 变更日期（施工日期）
- 变更内容（整改内容）
- 变更原因（整改原因）
- 影响范围（涉及范围）
- 关联 Issue（问题单编号）

## 产品经理判断方法

### 判断是否需要创建 Issue

类比装修施工的"是否需要施工问题单"：

| 情况 | 是否创建 Issue | 类比 |
|------|---------------|------|
| 新增需求 | 是 | 新增施工项目 |
| 需求逻辑变更 | 是 | 施工方案变更 |
| 需求优先级调整 | 是 | 施工顺序变更 |
| 需求取消 | 是 | 施工项目取消 |
| 文档格式调整 | 视重要性 | 施工日志格式调整 |

### 判断变更影响范围

类比装修施工的"影响哪些房间"：

| 影响维度 | 检查内容 | 类比 |
|---------|---------|------|
| **开发影响** | 是否需要修改代码？ | 是否需要重新施工？ |
| **测试影响** | 是否需要新增测试？ | 是否需要重新验收？ |
| **运营影响** | 是否影响上线推广？ | 是否影响入住时间？ |
| **数据影响** | 是否影响数据统计？ | 是否影响工程验收？ |
| **兼容性** | 是否影响旧版本？ | 是否影响已入住区域？ |

## 类比使用建议

### 推荐场景

| 节 | 类比使用 |
|---|---------|
| 第 3 节「核心概念解释」 | 文档修订类比（Git 核心概念） |
| 第 4 节「为什么产品经理容易误解」 | 装修施工类比（变更无记录的后果） |

### 类比对比

| 类比 | 优势 | 适用场景 |
|------|------|---------|
| **文档修订** | 直观、易懂、有版本感 | 解释 Git 核心概念 |
| **装修施工** | 强调记录、审核 | 解释 Issue/PR 流程 |
| **餐厅菜单** | 强调变更管理 | 解释变更追踪 |
