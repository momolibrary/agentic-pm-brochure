# 评审准备类比素材

> 知识点：B5-05 评审准备
> 素材类型：类比
> 来源：知乎专栏、人人都是产品经理

---

## 类比一：考试前复习清单

### 核心类比

**需求评审准备 = 考试前复习清单**

| 评审准备项 | 考试复习类比 |
|-----------|-------------|
| **需求背景** | 理解考试题目要求 |
| **数据支撑** | 背诵知识点、公式 |
| **方案完整** | 练习例题、覆盖题型 |
| **风险评估** | 预判难点题目 |
| **依赖确认** | 确认考试时间、地点、范围 |
| **演示材料** | 准备准考证、文具 |
| **预演方案** | 模拟考试、计时练习 |

### 类比说明

评审会就像考试现场，产品经理是考生，开发、测试、运营是考官。

准备不足就像考试前没复习，考官一问就答不上来。

准备充分就像考试前按清单复习，考官的问题都能应对。

### 常见追问类比

| 常见追问 | 考试类比 |
|---------|---------|
| "有数据支撑吗？" | 考官问"你的结论依据是什么公式？" |
| "竞品有这个功能吗？" | 考官问"有没有参考其他答案？" |
| "技术上能实现吗？" | 考官问"这个方法可行吗？" |

## 类比二：演讲前彩排

### 核心类比

**需求评审准备 = 演讲前彩排**

| 评审准备项 | 演讲彩排类比 |
|-----------|-------------|
| **需求背景** | 演讲主题、开场白 |
| **数据支撑** | 演讲素材、案例数据 |
| **方案完整** | 演讲内容、PPT结构 |
| **风险评估** | 预判观众提问 |
| **依赖确认** | 确认场地、设备、时间 |
| **演示材料** | PPT、道具、讲稿 |
| **预演方案** | 试讲、计时、调整节奏 |

### 类比说明

评审会就像演讲现场，产品经理是演讲者，开发、测试、运营是观众。

准备不足就像演讲前没彩排，上台就紧张忘词。

准备充分就像演讲前反复彩排，观众提问都能从容应对。

### 角色关注点类比

| 参会角色 | 观众类比 |
|---------|---------|
| **开发** | 技术专家，关注技术细节 |
| **测试** | 质量监督，关注边界条件 |
| **设计** | 视觉专家，关注体验一致性 |
| **运营** | 业务专家，关注上线效果 |

## 类比三：相亲前准备

### 核心类比

**需求评审准备 = 相亲前准备**

| 评审准备项 | 相亲准备类比 |
|-----------|-------------|
| **需求背景** | 自我介绍、家庭背景 |
| **数据支撑** | 学历、收入、房产证明 |
| **方案完整** | 未来规划、家庭计划 |
| **风险评估** | 预判对方问题 |
| **依赖确认** | 确认时间、地点、对方信息 |
| **演示材料** | 照片、证件、礼物 |
| **预演方案** | 想好话题、应对尴尬问题 |

### 类比说明

评审会就像相亲现场，产品经理是相亲者，开发、测试、运营是相亲对象和家人。

准备不足就像相亲前没准备，对方一问就尴尬。

准备充分就像相亲前充分了解对方，问题都能从容回答。

## 产品经理判断方法

### 判断准备是否充分

类比考试复习清单：

| 自检问题 | 考试类比 | 通过标准 |
|---------|---------|---------|
| 能用一句话说清楚需求背景吗？ | 能说清楚考试题目吗？ | 是 |
| 有具体数字的数据支撑吗？ | 背诵了公式和数据吗？ | 有 |
| PRD覆盖异常流程了吗？ | 练习了所有题型吗？ | 覆盖 |
| 技术风险标注了吗？ | 标注了难点题目吗？ | 标注 |
| 依赖方确认排期了吗？ | 确认考试时间和地点了吗？ | 确认 |
| 会前预演了吗？ | 模拟考试了吗？ | 预演 |

### 判断预演必要性

类比演讲彩排：

| 情况 | 是否预演 |
|------|---------|
| 第一次主导评审 | 是（首次演讲必须彩排） |
| 需求复杂度高 | 是（内容复杂需要练习节奏） |
| 参会人级别高 | 是（重要观众需要认真准备） |
| 需求简单、评审经验丰富 | 可省略 |
