## 类比一：流水线质检 — 人检 vs 机器检

### 类比场景

老李对小想说：

"你见过工厂流水线吗？以前质检靠人工，工人一个个看，速度慢、漏检多。后来机器质检，速度快、漏检少。但机器质检后，人干什么？人做'机器检什么'的决策。机器检测尺寸、重量、表面瑕疵，但什么叫'合格'？什么叫'瑕疵'？这些标准是人定的。机器只负责执行检测，人定检测标准。"

### 对应概念映射

| 工厂流水线场景 | 团队审查流程重构 |
|---------------|-----------------|
| 以前人工质检 | 以前人工审查 |
| 工人一个个看 | 审稿人一个个看 |
| 速度慢漏检多 | 审查节奏慢漏检多 |
| 后来机器质检 | 后来 AI 辅助审查 |
| 速度快漏检少 | 审查节奏快漏检少 |
| 机器检测尺寸重量瑕疵 | AI 检查格式逻辑术语 |
| 人定合格瑕疵标准 | 人定审查标准 |
| 机器执行检测人定标准 | AI 执行审查人定标准 |

### 类比要点

- **速度变化**：人工质检慢、机器质检快
- **漏检变化**：人工漏检多、机器漏检少
- **人做什么**：定检测标准
- **机器做什么**：执行检测

## 类比二：考试阅卷 — 人工阅卷 vs 系统阅卷

### 类比场景

老李说："高考阅卷以前全人工，老师一个个看作文，累得够呛，评分还有主观偏差。后来主观题还是人工阅，客观题系统阅。系统阅卷速度快、评分一致，老师只阅主观题。阅卷节奏快了，老师压力小了。但评分标准是谁定的？老师定的。系统只是执行评分标准。"

### 对应概念映射

| 考试阅卷场景 | 团队审查流程重构 |
|-------------|-----------------|
| 以前全人工阅卷 | 以前全人工审查 |
| 老师一个个看作文 | 审稿人一个个看文稿 |
| 累得够呛评分偏差 | 累压力大标准不一 |
| 后来客观题系统阅 | 后来格式逻辑 AI 检查 |
| 主观题还是人工阅 | 品味判断还是人工审 |
| 阅卷节奏快了 | 审查节奏快了 |
| 老师压力小了 | 审稿人压力小了 |
| 评分标准老师定 | 审查标准人定 |

### 类比要点

- **节奏变化**：人工阅卷慢、系统阅卷快
- **压力变化**：人工阅卷累、系统阅卷轻松
- **分工变化**：客观题系统、主观题人工
- **类比到审查**：格式逻辑 AI、品味判断人

## 类比三：医院体检 — 普检 vs 专家诊

### 类比场景

"你体检过吗？体检流程是这样的：先做一堆基础检查，血压、血常规、心电图，这些是普检，机器做、速度快、结果标准化。然后有问题了才找专家诊断。专家诊断是深度判断，需要人的经验直觉。体检节奏快，因为普检机器做。专家诊断精，因为人的判断深。AI 审查就像体检普检，先把基础问题筛出来，人才做深度判断。"

### 对应概念映射

| 医院体检场景 | 团队审查流程重构 |
|-------------|-----------------|
| 基础检查机器做 | 格式逻辑 AI 检查 |
| 血压血常规心电图 | 格式术语一致性 |
| 速度快结果标准化 | 速度快结果标准化 |
| 有问题找专家诊断 | 有品味问题人审 |
| 专家诊断是深度判断 | 品味判断是深度判断 |
| 需要人的经验直觉 | 需要人的品味直觉 |
| 普检机器诊断专家 | AI 基础人深度 |

### 类比要点

- **分层审查**：普检机器做，专家人做
- **节奏快**：普检机器做速度快
- **判断精**：专家人做判断精
- **类比审查**：基础 AI 检，深度人审

## 类比四：机场安检 — 普检 vs 复检

### 类比场景

"机场安检你熟吧？乘客先过普检，行李扫描、人过金属门，机器做速度快。有问题了才人工复检，安检员开箱检查。如果所有行李都人工开箱，安检队伍排十公里。普检机器做，复检人工做，节奏才能快。审查也一样，基础问题 AI 检，有问题才人工复检，节奏才能跟上产出量。"

### 对应概念映射

| 机场安检场景 | 团队审查流程重构 |
|-------------|-----------------|
| 普检机器做 | 基础 AI 检查 |
| 行李扫描金属门 | 格式逻辑术语 |
| 速度快 | 审查节奏快 |
| 有问题人工复检 | 有品味问题人审 |
| 安检员开箱检查 | 审稿人深度判断 |
| 全人工开箱排十公里 | 全人工审查跟不上产出 |
| 普检机器复检人工 | AI 基础人深度 |

### 类比要点

- **效率对比**：全人工安检排十公里，普检机器节奏快
- **类比产出量**：AI 产出量暴增，全人工审查跟不上
- **分层设计**：普检机器、复检人工
- **节奏重构**：普检快才能跟上客流

## 类比五：外卖配送 — 人工分拣 vs 系统调度

### 类比场景

"外卖平台你用过吧？以前人工分拣订单，调度员一个个看订单、分配骑手，速度慢、配错多。后来系统调度，算法自动分配骑手，速度快、配错少。但配送标准是谁定的？平台定的。什么叫'准时'？什么叫'最优路线'？这些标准是人定的。系统只是执行调度。审查也一样，AI 分配审查任务，人定审查标准。"

### 对应概念映射

| 外卖配送场景 | 团队审查流程重构 |
|-------------|-----------------|
| 以前人工分拣订单 | 以前人工分配审查任务 |
| 调度员一个个看 | 审稿主管一个个分配 |
| 速度慢配错多 | 分配慢任务错配 |
| 后来系统调度 | 后来 AI 分配审查任务 |
| 算法自动分配 | AI 自动分配 |
| 速度快配错少 | 分配快任务匹配准 |
| 配送标准平台定 | 审查标准人定 |
| 系统执行调度 | AI 执行分配 |

### 类比要点

- **调度变化**：人工分拣慢、系统调度快
- **错误变化**：人工配错多、系统配错少
- **人做什么**：定配送标准
- **系统做什么**：执行调度分配

## 审查流程重构要点

### 产出量变化

| 阶段 | 产出量 | 审查节奏 | 人力配置 |
|------|--------|---------|---------|
| 以前 | 产出量低 | 全人工审查能跟上 | 审稿人够用 |
| AI 引入后 | 产出量暴增 | 全人工审查跟不上 | 审稿人不够用 |
| 重构后 | 产出量暴增 | AI 辅助审查能跟上 | 审稿人够用（只审品味） |

### 审查标准变化

| 审查类型 | 以前 | 重构后 |
|---------|------|--------|
| 格式审查 | 人工审 | AI 检查 |
| 逻辑审查 | 人工审 | AI 检查 |
| 术语一致性 | 人工审 | AI 检查 |
| 品味判断 | 人工审 | 人工审 |
| 风格取舍 | 人工审 | 人工审 |
| 最终拍板 | 人工审 | 人工审 |

### 人做什么 AI 做什么

| 人做的事 | AI 做的事 |
|---------|---------|
| 定审查标准 | 执行格式检查 |
| 品味判断 | 执行逻辑检查 |
| 风格取舍 | 执行术语检查 |
| 最终拍板 | 自动分配审查任务 |
| 复检决策 | 标记问题点 |
| 标准更新 | 统计审查数据 |
