## IDE 是什么？

**IDE（Integrated Development Environment，集成开发环境）** 是开发写代码的核心工具。

它不是「写代码的记事本」，而是集成了多种能力的综合工作台：
- **代码编辑器**：语法高亮、代码补全、错误提示
- **调试器**：断点调试、变量查看、调用栈追踪
- **版本管理**：Git 集成，可视化提交历史
- **终端**：直接运行命令，无需切换窗口
- **AI 辅助**（新一代 IDE）：代码生成、智能补全、问题解答

---

## 为什么需要 IDE？

没有 IDE 之前，开发是这样的：
1. 用文本编辑器写代码
2. 切到命令行编译
3. 发现报错，切回编辑器找问题
4. 再切到命令行运行
5. 再切到浏览器看效果
6. …循环往复

有 IDE 之后：
- 所有操作在一个窗口完成
- 错误提示实时显示在代码旁边
- Git 操作可视化，不用记命令
- 调试时能直接看到变量值

**核心价值**：把分散的工具整合成统一工作流，减少「切窗口」的摩擦。

## IDE vs 文本编辑器

| 能力 | 文本编辑器（如记事本） | IDE |
|------|----------------------|-----|
| 语法高亮 | ❌ 或需插件 | ✅ 内置 |
| 代码补全 | ❌ | ✅ 智能补全 |
| 调试 | ❌ | ✅ 断点调试 |
| Git 管理 | ❌ 需命令行 | ✅ 可视化 |
| 终端集成 | ❌ | ✅ 内置终端 |
| 项目导航 | ❌ 手动找文件 | ✅ 文件树/搜索 |
| AI 辅助 | ❌ | ✅ 新一代 IDE |

## 产品经理视角的理解

**类比**：IDE 之于开发，就像 Figma 之于设计师。

| 设计师工具（Figma） | 开发工具（IDE） |
|-------------------|----------------|
| 设计稿编辑 | 代码编辑 |
| 组件库 | 代码库/模板 |
| 协作评论 | Git 提交记录 |
| 版本历史 | Git 分支/历史 |
| 插件扩展 | IDE 插件市场 |
| 输出切图/样式 | 输出编译产物 |

产品经理如果不理解 IDE，就像不理解设计师用 Figma 干什么一样——你不知道他们在哪个「工作台」上产出内容，协作时就容易产生沟通障碍。
