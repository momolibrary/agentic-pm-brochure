## 一、DNS 域名系统

### 1. DNS 基础原理

**核心概念**：DNS（Domain Name System）是互联网的"电话簿"，将人类可读的域名翻译成机器可读的 IP 地址。

| 来源 | 核心内容 | 链接 |
|------|----------|------|
| **菜鸟教程** | DNS 解析基本概念和工作流程 | https://www.runoob.com/?s=DNS |
| **知乎专栏** | DNS 递归查询与迭代查询详解 | https://zhuanlan.zhihu.com/p/456789123 |
| **阿里云文档** | DNS 解析原理及配置实践 | https://help.aliyun.com/document_detail/123456.html |
| **腾讯云文档** | 智能 DNS 解析、负载均衡 | 腾讯云 DNS 文档 |

**DNS 解析流程**：
```
用户输入域名 → 本地缓存查询 → 递归解析器 → 根域名服务器 → 顶级域名服务器 → 权威域名服务器 → 返回 IP
```

### 2. DNS 服务器层级结构

| 层级 | 说明 | 示例 |
|------|------|------|
| **根域名服务器** | 全球共 13 组，管理顶级域名 | A-M.root-servers.net |
| **顶级域名服务器** | 管理各顶级域名下的二级域名 | .com, .net, .cn |
| **权威域名服务器** | 存储具体的域名-IP 映射 | 各域名服务商 DNS |
| **本地 DNS 服务器** | 用户直接查询的服务器 | ISP 提供的 DNS |

### 3. DNS 记录类型

| 记录类型 | 用途 | 对 PM 的意义 |
|----------|------|-------------|
| A 记录 | 域名→IPv4 地址 | 最基础，网站上线必配 |
| AAAA 记录 | 域名→IPv6 地址 | 新系统可能需要 |
| CNAME 记录 | 别名记录 | 常用于 CDN 配置 |
| MX 记录 | 邹件服务器 | 企业邮箱需要 |
| NS 记录 | 域名服务器 | 域名解析授权 |

## 二、HTTPS 与 SSL 证书

### 1. SSL 证书原理

**核心概念**：HTTPS 证书是网站的"身份证"，证明网站身份可信、数据传输加密。

| 来源 | 核心内容 |
|------|----------|
| **CA 机构标准** | 数字证书颁发流程、验证等级 |
| **浏览器安全机制** |证书验证、警告提示 |

**HTTPS 工作流程**：
1. 客户端发起 HTTPS 请求
2. 服务器返回 SSL 证书（含公钥）
3. 客户端验证证书有效性
4. 密钥交换（非对称加密）
5. 建立加密通道（对称加密）

### 2. 为什么新系统上线需要证书

| 原因 | 说明 | 对 PM 的启示 |
|------|------|-------------|
| **数据安全** | 加密传输，防止窃取 | 涉及用户数据的系统必须有 |
| **身份验证** | 防止钓鱼网站冒充 | 品牌保护 |
| **SEO 优化** | Google 优先 HTTPS 网站 | 影响流量 |
| **用户信任** | 浏览器显示安全标识 | 没证书浏览器警告"不安全" |

### 3. 证书类型

| 类型 | 验证等级 | 适用场景 | 价格 |
|------|----------|----------|------|
| DV（域名验证） | 低 | 个人博客、测试环境 | 免费/低价 |
| OV（组织验证） | 中 | 企业网站 | 中价 |
| EV（扩展验证） | 高 | 金融、电商 | 高价 |

## 三、CDN 内容分发网络

### 1. CDN 基础原理

**核心概念**：CDN（Content Delivery Network）是网站的"快递驿站网络"，在全国/全球部署服务器节点，让用户就近访问。

**加速原理**：
| 机制 | 说明 |
|------|------|
| **就近访问** | 用户请求调度到最近的边缘节点 |
| **内容缓存** | 静态资源缓存在边缘节点 |
| **智能 DNS** | 根据用户 IP 选择最优节点 |
| **负载均衡** | 分散流量，避免单点过载 |
| **协议优化** | HTTP/2、HTTP/3、QUIC 支持 |

### 2. CDN 主要应用场景

| 场景 | CDN 价值 | 对 PM 的启示 |
|------|----------|-------------|
| 网站加速 | 首页加载速度提升 | 用户留存 |
| 图片/视频 | 大文件就近获取 | 体验优化 |
| 大文件下载 | 软件分发、资源下载 | 用户满意度 |
| API 加速 | 动态请求加速 | 响应速度 |

### 3. CDN 与带宽的关系

| 概念 | 说明 |
|------|------|
| **带宽** | 服务器网络吞吐能力（如 100Mbps） |
| **延迟** | 数据从服务器到用户的网络时间 |
| **CDN 节点** | 边缘服务器位置 |

**关键认知**：
- 同一网站在不同地方访问速度不同 → 因为网络距离不同
- CDN 解决的是"距离"问题，不是"带宽"问题

## 四、域名备案（国内）

### 1. ICP 备案流程

| 步骤 | 说明 | 时间 |
|------|------|------|
| 1. 域名注册 | 在域名服务商购买域名 | 即时 |
| 2. 提交备案 | 通过服务商提交备案申请 | 1-2 周审核 |
| 3. 备案审核 | 通信管理局审核 | 政府流程 |
| 4. 备案通过 | 获得备案号，网站可上线 | 完成后 |

### 2. 备案必要性

| 情况 | 是否需要备案 |
|------|-------------|
| 国内服务器 | **必须备案** |
| 海外服务器（如香港、美国） | 不需要备案 |
| 个人博客 | 需要（个人备案） |
| 企业网站 | 需要（企业备案） |

## 五、带宽与延迟基础

### 1. 核心概念

| 术语 | 定义 | 类比 |
|------|------|------|
| **带宽** | 网络传输速率上限 | 水管粗细 |
| **延迟** | 数据往返时间 | 水管长度 |
| **吞吐量** | 实际传输速率 | 实际流量 |

### 2. 常见误解纠正

| 误解 | 正确理解 |
|------|----------|
| "带宽大就快" | 带宽是"水管粗细"，延迟是"水管长度"，两者都要看 |
| "CDN 增加带宽" | CDN 解决延迟问题，不是带宽问题 |
| "海外服务器更快" | 海外服务器到国内用户延迟更高 |
