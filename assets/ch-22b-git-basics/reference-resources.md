## 二、概念解释来源摘要

### Git 基础概念（来源：Pro Git + 廖雪峰）

| 术语 | 定义 | 来源 |
|------|------|------|
| **Git** | 分布式版本控制系统，记录文件变更历史 | Pro Git 第一章 |
| **Commit（提交）** | 一次变更的快照，记录"改了什么、谁改的、什么时候" | Pro Git 第二章 |
| **Branch（分支）** | 指向提交对象的可变指针，独立开发线 | Pro Git 分支章节 |
| **Merge（合并）** | 把不同分支的变更合在一起 | Pro Git 分支章节 |
| **Conflict（冲突）** | Git 无法自动决定保留哪个版本，需人工干预 | Pro Git 分支章节 |

### 时光机类比来源

| 类比 | 来源 | 原文/出处 |
|------|------|----------|
| **时光机** | 廖雪峰教程 | "Git 允许你回到任意一个历史版本，就像时光机一样" |
| **游戏存档** | 通用类比 | 每个 commit 就像一个游戏存档点 |
| **快照/照片** | Pro Git | "commit 就像给项目拍一张照片" |
