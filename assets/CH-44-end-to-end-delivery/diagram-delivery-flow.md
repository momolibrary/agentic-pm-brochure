## 流程图要点标注

### 关键转折点

1. **需求收集 → 需求结构化**：从"一句话需求"变成"可执行需求"
2. **方案预演 → 技术评审**：从"发现问题"到"确认方案"
3. **测试验收 → 上线前检查**：从"功能正确"到"上线就绪"
4. **上线前检查 → 上线后跟踪**：从"上线成功"到"持续改进"

### 常见问题暴露点

| 阶段 | 常见问题 |
|------|---------|
| 需求结构化 | 异常流程遗漏、状态流转不完整 |
| 方案预演 | 边界条件未覆盖、技术难点未识别 |
| 测试验收 | bug根因判断错误、验收标准不清晰 |
| 上线前检查 | 监控未配置、回滚预案缺失 |
| 上线后跟踪 | 反馈响应不及时、问题根因未分析 |
