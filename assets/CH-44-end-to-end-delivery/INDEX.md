## 类比素材汇总

### 核心类比

1. **装修类比**（analogy-renovation.md）
   - 需求收集 → 业主提出装修需求
   - 需求结构化 → 设计师画设计图
   - 方案预演 → 施工前预演检查
   - 技术评审 → 与施工队确认方案
   - 开发实现 → 施工队干活
   - 测试验收 → 施工后验收检查
   - 上线前检查 → 交房前检查清单
   - 上线后跟踪 → 业主入住后反馈

2. **导演类比**（analogy-director.md）
   - 产品经理全程参与 → 导演从剧本到上映全程参与
   - 需求是蓝图不是终点 → 剧本是蓝图不是终点
   - 验收是产品经理责任 → 审片是导演责任
   - 上线是周密准备过程 → 上映是周密准备过程

3. **遗漏类比**（analogy-common-missing.md）
   - 异常流程遗漏 → 装修没考虑水管爆裂
   - 状态流转不完整 → 装修只画完成状态
   - 权限矩阵缺失 → 装修没写房间权限
   - 边界条件未覆盖 → 装修没考虑尺寸边界
   - 系统依赖未确认 → 装修没确认水电接口

## AI辅助场景汇总

| 交付阶段 | AI辅助环节 | 产品经理职责 |
|---------|-----------|-------------|
| 需求结构化 | 需求拆解、字段表生成 | 审核、补充、修正AI输出 |
| 方案预演 | 模拟技术方案 | 判断模拟结果是否符合业务场景 |
| 测试验收 | 测试用例生成 | 补充业务边界用例 |
| 文档评审 | 文档完整性检查 | 判断问题是否重要 |

### AI辅助局限

1. AI覆盖常规，遗漏业务边界 → 产品经理补充业务边界
2. AI输出需验证 → 产品经理审核修正
3. AI不懂业务背景 → 产品经理补充背景信息

## 必须纠正的误解素材

### 误解1："需求写完交给开发就完了"

**正确理解素材**：
- analogy-director.md：导演不能只写剧本，要全程参与
- story-full-delivery-process.md：9幕故事展示全程参与必要性

### 误解2："有了AI就不需要深入了解技术"

**正确理解素材**：
- ai-assisted-delivery.md：AI覆盖常规，但业务边界需产品经理补充
- AI是放大器，基础认知越扎实，AI辅助效果越好

### 误解3："只要功能做对了就算交付成功"

**正确理解素材**：
- story-full-delivery-process.md 第7幕：上线前检查清单
- analogy-renovation.md：装修成功 = 设计正确 + 施工正确 + 验收就绪 + 细节完整
- 交付成功 = 功能正确 + 异常覆盖 + 监控就绪 + 文档完整
