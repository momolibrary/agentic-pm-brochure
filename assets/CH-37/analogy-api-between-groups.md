# 类比：Human+Agent 组间沟通 = API 调用

> 知识点：B5-07
> 用途：第 6 节「产品经理应该如何做」

---

## 类比核心

**Human+Agent 组之间的沟通，就像两个服务通过 API 交互。**

- 传统团队沟通：打电话 → 必须双方同时在线，口头传达，信息易失真
- Human+Agent 组沟通：API 调用 → 不需要同时在线，结构化传输，可追溯

---

## 类比展开

### 打电话（传统团队沟通）

| 打电话的特点 | 传统团队沟通的问题 |
|------------|------------------|
| 必须双方同时在线 | 需要所有人同时在场（同步成本高） |
| 信息口头传达 | 信息靠口头同步，易失真 |
| 打完电话信息就没了 | 会议纪要整理费时间，信息难以追溯 |
| 一次只能和一个人通话 | 沟通效率低，多人协调靠开会 |

**关键问题**：打电话是「同步沟通」，效率低、信息易失真、难以追溯。

### API 调用（Human+Agent 组沟通）

| API 调用的特点 | Human+Agent 组沟通的优势 |
|--------------|------------------------|
| 不需要同时在线 | 异步沟通，随时响应 |
| 信息结构化传输 | 文档/Issue/API 格式化，信息清晰 |
| 请求和响应可追溯 | 所有信息在 Issue/文档中可查 |
| 一个请求可触发多个响应 | Agent 自动解析、分发、执行 |

**关键优势**：API 调用是「异步沟通」，效率高、信息清晰、可追溯。

---

## 类比对比

### 场景：确认产品需求变更

| 传统方式（打电话） | Human+Agent 方式（API 调用） |
|------------------|----------------------------|
| 1. 发邮件约会议时间 | 1. 创建 Issue，描述变更内容 |
| 2. 开会讨论 30 分钟 | 2. Agent 自动解析需求，生成任务 |
| 3. 会议纪要整理 | 3. 另一组 Agent 接收任务，开始执行 |
| 4. 邵件确认纪要 | 4. Human 只需审核结果 |
| 5. 信息遗漏，再开会 | 5. 所有信息在 Issue 中可追溯 |

**效率对比**：
- 传统方式：至少 4 步，每步可能出错，会议占大量时间
- Human+Agent 方式：Human 只需创建 Issue + 审核结果，中间全自动化

---

## 类比应用场景

### 场景一：跨部门协作

传统：研发 ↔ 产品 ↔ 运营，每周 3 次会议
Human+Agent：研发组 ↔ 产品组 ↔ 运营组，Issue 协作，无会议

### 场景二：信息同步

传统：信息靠会议同步，一人缺席信息断链
Human+Agent：信息在 Issue/文档中，随时可查，无需在场

---

## 类比局限

### 什么时候还是需要「打电话」？

| 需要打电话（会议） | 可以用 API（Issue/文档） |
|------------------|------------------------|
| 战略讨论、重大决策 | 信息同步、任务分配 |
| 关键客户面对面 | 状态汇报、进度对齐 |
| 创意发散、头脑风暴 | 规格确认、需求澄清 |
| 关系建立、信任培养 | 执行跟进、问题排查 |

**关键原则**：能用接口解决的，别开会。

---

## 类比总结

**一句话记住这个类比**：

> 传统团队沟通是打电话，同步低效；Human+Agent 组沟通是 API 调用，异步高效。
> 能用接口解决的，别开会。
