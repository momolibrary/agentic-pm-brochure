## 图示需求

### 需求描述

一张「传统接力模式 vs Agentic Design协作模式」的对比示意图

### 图示内容建议

**左侧：传统接力模式（接力棒）**

```
设计师 ──设计稿──→ 产品经理 ──需求──→ 开发 ──实现──→ 产品验收 ──问题──→ 设计师修改
    ↑                                                        │
    └────────────────────────────────────────────────────────┘
                          循环多次，耗时两周
```

标注：异步传递、信息损耗、来回反复、效率低下

**右侧：Agentic Design协作模式**

```
设计师 + 产品经理 + 开发
        ↓
    AI画布（实时生成）
        ↓
    当场讨论（技术限制、设计细节）
        ↓
    当场确认
        ↓
    一次定稿（2小时）
```

标注：同步协作、信息无损、实时决策、效率提升

**中间：连接线和说明**

- 「从两周 → 两小时」
- 「从异步传递 → 同步协作」
- 「从来回反复 → 当场定稿」

### 图示风格建议

- 使用对比布局：左vs右
- 使用颜色区分：传统模式=灰色（低效）、新模式=绿色（高效）
- 使用时间对比：标注「两周」vs「两小时」
- 整体简洁清晰，适合PM理解
