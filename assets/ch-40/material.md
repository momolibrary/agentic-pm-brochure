## 核心概念解释

### 什么是设计系统？

设计系统是一个**完整的、可复用的设计标准、模式和组件集合**，团队用它来创建一致的用户界面。它是设计和开发团队的**单一事实来源（Single Source of Truth）**。

核心组成要素：

| 组成部分 | 内容 | 作用 |
|---------|------|------|
| **设计原则** | 核心价值观、设计哲学 | 指导所有设计决策 |
| **UI Kit** | 预制界面元素（设计稿） | 快速原型设计 |
| **组件库** | 可复用代码组件 | 快速开发实现 |
| **设计规范** | 使用指南、间距规则、响应式行为 | 确保正确使用 |
| **品牌素材** | Logo、图标、插画、品牌色 | 品牌一致性 |
| **文档** | 使用指南、最佳实践、示例 | 知识传承 |

### UI Kit 与设计系统的区别

| 维度 | UI Kit | 设计系统 |
|------|--------|---------|
| **内容范围** | 只有设计素材 | 包含设计、代码、规范、文档 |
| **使用者** | 主要给设计师 | 设计师 + 开发 + 产品 |
| **层级** | 设计系统的一部分 | 更大的体系 |
| **比喻** | 像「零件盒」 | 像「工厂生产规范手册 + 零件盒」 |

> **关键区分**：UI Kit 是设计系统的子集。设计系统不仅有组件，还有「如何使用这些组件」的规范和文档。

## 小团队起步建议

### 从小做大的路径

| 阶段 | 内容 | 时间投入 |
|------|------|---------|
| **阶段一** | 统一色板、字体、按钮样式 | 1-2 周 |
| **阶段二** | 建立基础组件库（表单、卡片、导航） | 2-4 周 |
| **阶段三** | 添加设计规范文档和使用指南 | 持续迭代 |
| **阶段四** | 完善组件库、接入代码实现 | 按需扩展 |

### 判断框架（4 个问题）

1. 产品方向是否足够稳定（6+ 个月承诺）？
2. 是否有 2-3 个反复使用的 UI 模式？
3. 未来一年是否会显著扩大团队规模？
4. 设计/开发摩擦是否已成为可衡量的问题？

**如果回答「是」超过 3 个**，可能适合开始轻量级设计系统。
