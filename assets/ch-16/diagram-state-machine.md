## 图示三：并发冲突时序图

### 两人同时审批同一订单

```
时间轴        用户A                    订单状态                  用户B
──────────────────────────────────────────────────────────────────────
T0            读取状态(待审批)          待审批                    读取状态(待审批)
               │                        │                        │
T1            判断可审批                待审批                    判断可审批
               │                        │                        │
T2            写入审批记录              待审批                    写入审批记录
               │                        │                        │
T3            更新状态(已通过)          已通过                    更新状态(已驳回) ← 冲突！
               │                        │                        │
T4            ────────────────────────→ 已驳回 ←─────────────────────
               │                        │                        │
               ▼                        │                        ▼
            审批成功但状态变了           │                    审批成功但状态变了

问题：两个请求都认为自己成功，但状态被后一个请求覆盖，结果不一致
```

### 加锁后的处理

```
时间轴        用户A                    订单状态                  用户B
──────────────────────────────────────────────────────────────────────
T0            申请锁                   待审批                    申请锁
               │                        │                        │
T1            获得锁 ✓                 待审批                    等待锁...
               │                        │                        │
T2            读取状态(待审批)          待审批                    等待锁...
               │                        │                        │
T3            判断可审批                待审批                    等待锁...
               │                        │                        │
T4            更新状态(已通过)          已通过                    等待锁...
               │                        │                        │
T5            释放锁                    已通过                    等待锁...
               │                        │                        │
T6            ────────────────────────── 已通过 ←───────────────────获得锁 ✓
               │                        │                        │
               ▼                        │                        ▼
            审批成功 ✓                  │                    读取状态(已通过)
                                     已通过                    判断不可审批
                                     已通过                    提示"订单已处理"

结果：用户B等待用户A完成后才操作，发现状态已变，提示冲突
```
