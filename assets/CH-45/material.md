## 核心知识点素材

### C1-01 审批流程设计要素清单

| 要素 | 说明 | 设计要点 |
|------|------|----------|
| **审批角色** | 谁有审批权 | 按角色、按金额阈值、按部门层级 |
| **审批层级** | 单级还是多级 | 条件分支：金额超过阈值需要更高级审批 |
| **审批规则** | 什么情况触发审批 | 条件触发：金额、类型、紧急程度 |
| **驳回处理** | 驳回后能做什么 | 可重新提交、可申诉、可修改后提交 |
| **撤回机制** | 申请人能否撤回 | 待审批状态时可撤回，已通过不可撤回 |
| **超时处理** | 审批超时怎么办 | 自动通过、自动驳回、升级催办 |
| **并行会签** | 多人同时审批 | 所有人通过才算通过，一人驳回即驳回 |
| **串行审批** | 按顺序审批 | 上一级通过才能到下一级 |
| **状态机设计** | 状态流转完整定义 | 状态、转换条件、异常处理 |

### C1-02 权限系统设计六步法

| 步骤 | 内容 | 输出 |
|------|------|------|
| **1. 识别角色** | 列出系统所有用户角色 | 角色清单（如：管理员、部门经理、普通员工、审计员） |
| **2. 定义功能权限** | 每个角色能执行什么操作 | 功能权限表（角色 × 操作 = Yes/No） |
| **3. 定义数据权限范围** | 每个角色能看到哪些数据 | 数据权限表（角色 × 数据范围 = 全部/本部门/本人/无） |
| **4. 设计权限继承规则** | 角色之间有无继承关系 | 继承规则（如：经理继承员工权限 + 部门管理权限） |
| **5. 处理特殊角色** | 临时角色、代理角色 | 特殊角色定义（临时授权、代理审批） |
| **6. 输出权限矩阵** | 整合所有权限定义 | 权限矩阵文档（角色 × 功能 × 数据） |

## 常见遗漏清单

### 审批流程常见遗漏

| 遗漏项 | 问题 | 修复建议 |
|--------|------|----------|
| **超时处理** | 审批卡住不动 | 设置超时自动升级或催办 |
| **撤回机制** | 申请人想撤回但没入口 | 待审批状态允许撤回 |
| **驳回后动作** | 驳回后不知道能做什么 | 明确：可重新提交、可申诉、可修改 |
| **会签逻辑** | 多人审批时逻辑不清 | 明确：全会签通过才算通过 |
| **审批记录** | 没有审批历史可追溯 | 记录每次审批动作、时间、意见 |
| **条件分支** | 不同情况走不同审批路径 | 按金额、类型、紧急程度分支 |

### 权限系统常见遗漏

| 遗漏项 | 问题 | 修复建议 |
|--------|------|----------|
| **数据权限边界** | 只考虑功能权限，忽略数据范围 | 定义：能看到全部、本部门、本人、无 |
| **权限继承** | 新角色从哪里继承权限 | 明确继承规则，避免权限冗余或缺失 |
| **临时授权** | 临时需要额外权限没机制 | 设计临时授权流程，过期自动回收 |
| **权限变更审计** | 不知道谁改了权限 | 记录权限变更历史 |
| **角色冲突** | 一人多角色时权限叠加逻辑 | 明确：权限叠加还是取最高 |
| **敏感操作二次确认** | 删除等高风险操作无保护 | 添加二次确认或审批 |
