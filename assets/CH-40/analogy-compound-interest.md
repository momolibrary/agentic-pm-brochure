# 类比：设计系统 = 复利投资

> 知识点：B6-05 — 设计系统
> 类比类型：价值解释型类比
> 适用章节：第 3 节（核心概念）、第 5 节（复利效应）

---

## 类比核心

**搭建设计系统就像做复利投资。**

前期投入一笔资金（时间、精力），后期每次使用都在「赚取利息」，收益随时间指数级增长。

---

## 类比展开

### 投资阶段对比

| 设计系统 | 复利投资 | 对应关系 |
|---------|---------|---------|
| **前期投入** | 初始本金投入 | 花时间建立色板、组件、规范 |
| **每次复用** | 每期利息 | 每次使用组件节省设计/开发时间 |
| **收益累积** | 利滚利 | 新设计稿自动套用系统风格 |
| **长期回报** | 指数级增长 | 团队效率持续提升 |

### 具体场景映射

**没有设计系统（没有投资）：**

```
每次设计新页面：
  → 重新定义按钮样式（浪费时间）
  → 和开发确认间距（沟通成本）
  → 品牌不一致（用户体验下降）
  → 就像每次消费都要重新挣工资
```

**有设计系统（有投资）：**

```
每次设计新页面：
  → 直接用现有组件（节省时间）
  → 规范已定义，无需再确认（沟通成本低）
  → 品牌一致（用户体验好）
  → 就像每月有固定利息收入
```

### 时间收益曲线

```
时间轴：      第1月    第3月    第6月    第12月    第24月
无设计系统：  [======] [======] [======] [======]  [======]  ← 每次付出相同
有设计系统：  [=====]  [====]   [===]    [==]      [=]       ← 付出递减

初期付出高（建立系统），后期付出低（复用组件），总收益随时间累积。
```

---

## 数据支撑

行业经验表明：

- **设计系统上线后 6-12 个月**开始看到明显的效率提升
- **组件复用率**：成熟系统可达 80%+
- **设计/开发摩擦减少**：沟通确认时间可减少 50%

---

## 类比价值

### 为什么用这个类比？

1. **PM 理解金融概念**：复利是产品经理熟悉的商业概念
2. **直观展示时间价值**：强调「前期投入换来后期收益」的核心逻辑
3. **反驳「太重」误解**：不是「太重」，而是「投资回报周期」

### 类比边界

| 适用 | 不适用 |
|------|--------|
| 解释为什么要投入 | 解释具体技术实现 |
| 展示长期价值 | 展示短期成本 |
| 对比有无系统的差异 | 详细组件设计方法 |

---

## 可能的质疑

### 质疑：复利是确定性收益，设计系统不一定成功

**回应**：类比的核心是「前期投入换取后期复用价值」的逻辑，不是数学上的确定性。设计系统确实有失败风险（如维护不当），但成功案例的收益模式符合「复利」特征。

### 质疑：小团队没那么多设计需求，收益不明显

**回应**：复利投资可以从小额开始。统一色板和按钮也是起步投资，随团队成长收益自然累积。
