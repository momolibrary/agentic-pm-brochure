## 图示需求

### 需求描述

两张图示：
1. 服务蓝图三层结构示意图
2. 商业模式画布九格布局图

### 图示内容建议

**服务蓝图三层结构**

```
┌─────────────────────────────────────────────────────────────────┐
│                    服务蓝图三层结构                               │
├─────────────────────────────────────────────────────────────────┤
│ 用户行为层     ──→ 用户的具体行为步骤                              │
│                  （点击、浏览、输入等）                            │
│                    ← 互动线 →                                     │
├─────────────────────────────────────────────────────────────────┤
│ 前台交互层     ──→ 用户直接可见的触点                              │
│                  （界面、客服对话等）                              │
│                    ← 可视线 →                                     │
├─────────────────────────────────────────────────────────────────┤
│ 后台支撑层     ──→ 用户看不见但支撑前台的活动                       │
│                  （系统处理、数据查询等）                           │
│                    ← 内部线 →                                     │
├─────────────────────────────────────────────────────────────────┤
│ 内部流程层     ──→ 组织内部的基础支撑流程                          │
│                  （IT运维、财务、HR等）                            │
└─────────────────────────────────────────────────────────────────┘
```

标注要点：
- 三条分割线的含义（互动线、可视线、内部线）
- 前台用户可见，后台用户不可见
- 用户旅程图只覆盖用户行为层+前台交互层

**商业模式画布九格布局**

```
┌─────────────┬─────────────┬───────────────┬─────────────┬──────────────────┐
│             │             │               │             │                  │
│  重要伙伴   │  关键活动   │   价值主张    │  客户关系   │   客户细分       │
│             │             │               │             │                  │
│             ├─────────────┤               ├─────────────┤                  │
│             │             │               │             │                  │
│             │  核心资源   │               │  渠道通路   │                  │
│             │             │               │             │                  │
├─────────────┴─────────────┴───────────────┴─────────────┴──────────────────┤
│                                                                            │
│                        成本结构                收入来源                    │
│                                                                            │
└────────────────────────────────────────────────────────────────────────────┘
```

标注要点：
- 左侧（供给）：如何创造价值
- 右侧（客户）：为谁创造价值、如何交付
- 底部（财务）：成本和收入
- 中间：价值主张是核心

### 图示风格建议

- 使用分层/分格布局，清晰区分各部分
- 使用颜色区分：前台用蓝色，后台用灰色，内部用浅色
- 添加分割线标注（可视线、内部线）
- 整体简洁清晰，适合 PM 理解

## 验收标准对照

### 必须回答的问题

1. **服务蓝图和用户旅程图的区别？**
   - 素材支撑：`concept-service-blueprint.md`（对比表格）
   - 类比支撑：`analogy-service-blueprint-restaurant.md`（顾客旅程 vs 餐厅全景）
   - 关键结论：旅程图只看前台，蓝图还看后台支撑和内部操作

2. **商业模式画布九格？**
   - 素材支撑：`concept-business-model-canvas.md`（九要素详解）
   - 类比支撑：`analogy-business-model-canvas-tea-shop.md`（奶茶店九要素）
   - 九要素：客户细分、价值主张、渠道、客户关系、收入来源、核心资源、关键活动、重要伙伴、成本结构

3. **为什么产品经理需要理解商业模式？**
   - 素材支撑：`story-product-manager-misunderstanding-business.md`（功能优先级争论）
   - 类比支撑：`analogy-business-model-canvas-tea-shop.md`（奶茶店功能决策）
   - 关键结论：功能设计和优先级排序必须服务于商业模式

### 读者能做到的事

1. **能理解服务蓝图的前后台视角**
   - 素材支撑：餐厅类比（前台餐桌、后台厨房）
   - 验证方式：能解释功能设计要考虑后台支撑

2. **能用商业模式画布做基本的商业模式分析**
   - 素材支撑：奶茶店类比（九要素对应开店问题）
   - 验证方式：能用画布框架分析一个产品的商业模式

### 必须纠正的误解

1. **误解："产品经理只管功能，商业模式是老板的事"**
   - 素材支撑：`story-product-manager-misunderstanding-business.md`（功能优先级争论）
   - 正确理解：产品决策必须服务于商业模式
