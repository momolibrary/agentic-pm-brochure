## 图示需求

### 需求描述

两张示意图：
1. MVC 分层示意图
2. CQRS 读写分离示意图

### MVC 分层示意图建议

**左侧：用户请求流程**

```
用户 → Controller → Model → 数据库
              ↓
            View → 用户看到界面
```

**右侧：各层职责说明**

| 层 | 职责 | 餐厅类比 |
|---|------|---------|
| Model | 数据存储、业务逻辑 | 厨房 |
| View | 数据展示、用户界面 | 菜单/摆盘 |
| Controller | 请求处理、协调 Model 和 View | 服务员 |

**标注要点**：
- 「改显示样式 = 只改 View」
- 「改数据内容 = 改 Model + Controller」

### CQRS 读写分离示意图建议

**左侧：传统方式（读写同一通道）**

```
用户请求（读或写）→ 单一系统 → 数据库
```

标注：读写混在一起，高峰时互相阻塞

**右侧：CQRS 方式（读写分离）**

```
用户写请求 → Command 通道 → 写模型 → 数据库
用户读请求 → Query 通道 → 读模型（缓存）
```

标注：读写分离，各自优化、互不影响

**类比标注**：
- 「写 = 柜台存取款，严谨通道」
- 「读 = App 查余额，便捷通道」

1. **能理解 MVC 分层架构**

3. **能在开发提出 CQRS 时理解其适用场景**
