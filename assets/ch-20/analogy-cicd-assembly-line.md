# 类比素材：CI/CD = 工厂流水线质检

> 知识点：A7-04 DevOps 与持续交付
> 类比核心：CI/CD = 工厂流水线质检（每个环节都有质检站，不合格不放行）
> 更新日期：2026-04-19

---

## 类比核心

**CI/CD 就像工厂流水线的质检系统。**

每个环节都有质检站，不合格不放行。

---

## 详细类比

### 工厂流水线 vs CI/CD Pipeline

| 工厂流水线概念 | CI/CD 对应 | 说明 |
|---------------|-----------|------|
| **原料入库** | Git Commit/Push | 代码提交到仓库 |
| **原料检验** | Build + Unit Test | 确保代码能构建、基础逻辑正确 |
| **加工组装** | Integration Test | 确保模块能正确协作 |
| **半成品检验** | Deploy to Staging | 在接近生产的环境验证 |
| **成品检验** | UAT 测试 | 人工确认功能符合预期 |
| **出厂检验** | Pre-prod 环境 | 最后一道关卡 |
| **产品上架** | Deploy to Production | 对用户可见 |

### 质检站概念

**工厂流水线的特点：每个环节都有质检站。**

```
原料 → 质检站1 → 加工 → 质检站2 → 组装 → 质检站3 → 成品 → 质检站4 → 出厂
```

**CI/CD Pipeline 的特点：每个阶段都有验证。**

```
代码 → Build → Unit Test → Integration Test → Staging → UAT → Pre-prod → Production
```

---

### 为什么不能跳过质检站

**工厂案例**：

> 「假设你是工厂质检员，有人告诉你「只改了一个零件，不用检」。你敢放行吗？」
>
> 「不敢。因为改一个零件可能影响整台设备。之前有个工人只换了一颗螺丝，结果螺丝型号错了，整台设备运行时螺丝松动，机器报废。」

**CI/CD 案例**：

> 「同样，改一行文案也可能影响整个系统。之前有个实习生「只改文案」，结果把变量名也改了，上线后整个页面空白。」

---

### 质检站的作用

| 质检站 | 工厂作用 | CI/CD 作用 |
|-------|---------|-----------|
| 质检站1（原料检验） | 确保原料合格，不合格拒收 | 确保代码能构建，构建失败不继续 |
| 贬检站2（加工检验） | 确保加工正确，有问题停工 | 确保基础逻辑正确，测试失败不继续 |
| 质检站3（组装检验） | 确保组装正确，有问题返工 | 确保模块协作正确，集成失败不继续 |
| 质检站4（成品检验） | 确保成品合格，不合格不出厂 | 确保功能正确，验证失败不上线 |

## 类比使用方式

### 产品经理容易理解的表述

**核心类比**：

> 「CI/CD 就像工厂流水线的质检系统。你的代码改动从提交到上线，每个环节都有质检站。质检站不合格，产品不放行。」

**补充类比**：

> 「你改了一行文案，以为「小改动不用检」。但工厂里改一颗螺丝也要走全流程质检。因为一颗螺丝可能影响整台设备，一行文案也可能影响整个系统。」

### 针对误解的类比纠正

**误解：「小改动不需要走发布流程」**

类比纠正：

> 「工厂流水线没有「小改动免检」这个概念。改一颗螺丝也要过所有质检站。同样，CI/CD 也没有「小改动免检」。任何改动都要走全流程。」

**误解：「为什么要等两天」**

类比纠正：

> 「质检是需要时间的。工厂里质检员检验一台设备可能要半天，CI/CD 里自动测试、人工验证也需要时间。这不是拖延，是质量控制。」

---

### 灰度发布的类比

**产品上架后的类比**：

> 「产品上架也不是一次性全上架。新品先上架少量，观察市场反应，再逐步铺开。」
>
> 「这就是灰度发布。先给 1% 用户用新版本，观察没问题再给更多用户。」

**对比**：

| 产品上架 | 灰度发布 |
|---------|---------|
| 先上架少量试销 | 先给 1% 用户使用 |
| 观察市场反应 | 观察系统运行 |
| 反应好再铺开 | 没问题再扩大范围 |
| 反应不好可撤回 | 有问题可回滚 |

---

## 类比的局限性

### 不完全对应的部分

| 工厂流水线 | CI/CD Pipeline | 差异说明 |
|-----------|---------------|---------|
| 物理产品 | 软件产品 | 软件可以快速迭代 |
| 人工质检为主 | 自动化为主 | CI/CD 大部分是自动测试 |
| 质检后产品不变 | 上线后可继续优化 | 软件可以持续迭代 |

### 需要补充说明

类比之后，补充：

> 「CI/CD 比工厂流水线更自动化。大部分质检是机器自动完成，只有最后的人工验证需要人确认。而且软件上架后还可以继续优化，不像工厂产品出厂就定型。」
