# standards/CLAUDE.md — 质量标准体系

> 父级：[../CLAUDE.md](../CLAUDE.md)

## 概述

本目录包含项目的全部质量标准，是所有内容创建和审核的**强制基准**。这些标准对全项目生效，任何书稿内容、Agent 输出均须遵守。

## 文件清单

| 文件 | 用途 | 何时加载 |
|------|------|---------|
| `style-guide.md` | 文风指南：核心原则、句长限制、幽默控制、人物一致性、10 要素文风 | 写作 / 审核时 |
| `term-glossary.md` | 术语表：标准译名、定义、类比说明、禁用表达 | 写作 / 审核 / 术语查询时 |
| `lint-rules.yaml` | 自动检查规则：文风 / 术语 / 结构三类检查项及阈值 | 执行 lint 检查时 |

## 关键约束摘要

以下为高频引用的核心约束，避免每次加载全文：

| 约束 | 值 | 来源 |
|------|-----|------|
| 句长上限 | ≤ 30 字 | style-guide.md |
| 幽默等级 | 1-3（不超过 3） | style-guide.md |
| 禁用空话词 | 赋能、抓手、闭环、沉淀、对齐 等 | style-guide.md / term-glossary.md |
| 老李人设 | 资深工程师，≥10 年经验，耐心但直接 | term-glossary.md |
| 小想人设 | 产品经理，2-3 年经验，聪明但缺工程常识 | term-glossary.md |
| 章节结构 | 10 要素（标题→引入场景→概念引出→核心讲解→类比解释→案例→常见误区→实操建议→延伸思考→章节小结） | style-guide.md |

## 三文件联动关系

```
style-guide.md ←→ lint-rules.yaml
     ↕                  ↕
term-glossary.md ←——————┘
```

- `lint-rules.yaml` 是 `style-guide.md` 的可执行化表达
- `term-glossary.md` 为两者提供术语基准
- 修改任一文件，须检查另外两个是否需同步更新

## 修改规则

1. **术语表变更**：新增/修改术语后，须全项目检查术语一致性（尤其已有章节）
2. **Lint 规则变更**：修改阈值或新增规则后，须同步更新 `style-guide.md` 对应说明
3. **文风指南变更**：影响面最广，须评估已有章节是否需要调整
4. **任何变更**：更新本文件的「关键约束摘要」如有涉及
