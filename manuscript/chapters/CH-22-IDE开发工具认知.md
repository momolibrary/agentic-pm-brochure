# CH-22 IDE — 开发工具认知

## 1. 本章要解决什么问题

IDE 是什么？为什么建议产品经理也装一个？

很多产品经理看到开发屏幕上花花绿绿的界面，觉得挺酷的，但不知道那是干什么的。你不需要学会写代码，但需要知道 IDE 是什么——就像你知道 Figma 是什么一样。

理解 IDE，你才能知道开发在哪个「工作台」上干活。

## 2. 真实业务场景开场

周一上午，小想第一次走进开发团队办公室。

路过一个开发工位，她看到屏幕上花花绿绿的界面：「这是什么软件？挺酷的。」

开发：「这是我的 IDE，写代码的地方。左边是代码，颜色不一样是语法高亮；右边是终端；下面是 Git 状态。」

小想：「哦，原来不是记事本啊。」

开发：「记事本只能写字。这东西能写、能调试、能运行、还能管版本——是开发的「工作台」。」

小想点点头。原来开发也有自己的「Figma」。

## 3. 核心概念解释

### IDE 是什么？

**IDE（集成开发环境）** 是开发写代码的核心工具，集成了多种能力：

- **代码编辑器**：语法高亮、代码补全
- **调试器**：断点调试、变量查看
- **版本管理**：Git 集成，可视化历史
- **终端**：直接运行命令
- **AI 辅助**：代码生成、智能补全（新一代 IDE）

### 为什么需要 IDE？

没有 IDE，开发要来回切换窗口：编辑器写代码、命令行编译、浏览器看效果。

有 IDE，所有操作在一个窗口完成。错误实时提示，Git 可视化。

**核心价值**：整合工具，减少「切窗口」的摩擦。

### 类比：IDE 之于开发，就像 Figma 之于设计师

| Figma（设计师） | IDE（开发者） |
|----------------|---------------|
| 设计稿编辑 | 代码编辑 |
| 组件库 | 代码库/模板 |
| 协作评论 | Git 提交记录 |
| 版本历史 | Git 分支/历史 |
| 插件扩展 | IDE 插件市场 |
| 输出切图/样式 | 输出编译产物 |

产品经理不理解 IDE，就像不理解设计师用 Figma 干什么一样。你不知道他们在哪个「工作台」产出内容，协作就容易产生障碍。

### VS Code、JetBrains、Cursor 有什么区别？

| IDE | 定位 | 核心特点 |
|-----|------|----------|
| **VS Code** | 免费、轻量 | 几秒打开，插件丰富 |
| **JetBrains** | 专业、收费 | 代码分析最强，$149/年 |
| **Cursor** | AI 原生 | AI 理解整个代码库 |

**VS Code**：微软出品，免费。适合前端、Python 开发。

**JetBrains**：专业 IDE，适合 Java/Kotlin 企业级开发。

**Cursor**：AI 原生 IDE，产品经理可用来写需求、画原型。

## 4. 为什么产品经理容易误解

### 误解：IDE 就是「写代码的记事本」

很多产品经理看到开发用 IDE，心想：不就是写代码的软件吗？和记事本差不多吧？

这个误解的问题在于：你没看到 IDE 的「集成」能力。

记事本只能「写字」，IDE 是「写字 + 调试 + 运行 + 管理 + AI 辅助」的综合台。

类比：
- 记事本 = 只能写字的白纸
- IDE = 有格子、有公式栏、有图表工具、能自动计算、能保存版本的 Excel

**不是「写」，是「写 + 做 + 管」。**

### 误解原因

多数产品经理没写过代码，看不到「调试」「版本管理」功能。只看到开发在「敲字」，就以为那是「写字」工具。

就像只看到设计师在 Figma 里「画图」，没看到组件库、版本对比、协作评论——就以为它只是画图软件。

## 5. 开发视角如何看这个问题

开发看 IDE，想的不是「有个软件写代码」，而是：

- 代码怎么组织？文件树要清晰
- 调试怎么跑？断点、变量、调用栈
- 版本怎么管？Git 提交、分支、合并
- 命令怎么跑？终端集成，不用切窗口
- AI 怎么用？补全、生成、问题解答

这些才是开发每天打交道的东西。

开发选择 IDE，考虑的是：

- **语言支持**：这个 IDE 支持我用的语言吗？
- **性能**：大项目打开快不快？
- **调试能力**：调试功能强不强？
- **AI 集成**：AI 辅助好不好用？

产品经理知道这些，就能理解开发为什么说「这个 IDE 不适合这个项目」。

## 6. 产品经理应该如何做

### 建议：装一个 Cursor

你不用写代码，但可以体验 AI 协作。

步骤：

1. 下载 Cursor（免费可用，Pro $20/月）
2. 打开侧边栏 AI 聊天窗口
3. 输入：「帮我写一个登录页需求文档。」

小想试了一下，发现 AI 生成的文档格式符合团队模板。

老李：「AI 时代的 IDE，PM 也能用。」

### 场景推荐

| 场景 | 推荐 |
|------|------|
| 想体验开发工具、零成本 | VS Code（免费） |
| 想用 AI 写需求、验证原型 | Cursor（AI 原生） |
| 公司有 JetBrains、想跟开发统一工具 | JetBrains |
| 不写代码，只是了解开发工作 | 不用装，知道 IDE 是什么就行 |

## 7. AI 可以如何辅助

### 在 Cursor 中用 AI

- **写需求文档**：描述需求，AI 生成结构化文档
- **画原型代码**：描述页面，AI 生成 HTML/CSS 代码
- **验证想法**：描述逻辑，AI 帮你推演实现方案

### 注意边界

AI 生成的是「草稿」，你需要：

- 检查格式是否符合团队规范
- 补充遗漏的业务细节
- 验证逻辑是否完整

老李说：「AI 是工具，判断力是你的。工具帮你提速，你把控质量。」
