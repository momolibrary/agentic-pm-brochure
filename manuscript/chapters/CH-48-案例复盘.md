# CH-48 案例复盘 — 遗漏复盘、翻车复盘与成功复盘

## 一、三种复盘，三种痛

周一上午，复盘会。

小想刚做完一个项目，上线后出了三个问题。

第一个：用户注销后，历史订单页面报错。需求里漏了"注销前检查订单状态"这个场景。老李问："漏在哪个环节？为什么漏？"小想说："需求梳理时只写了正常流程，没想异常分支..."

第二个：积分兑换功能上线第二天，用户投诉积分被清零。需求变更后只通知了测试，开发按旧规则写代码。老李问："根因是什么？是技术问题还是沟通问题？"小想愣住："好像是沟通问题？"

第三个：一个项目按时高质量上线。老李说："成功了也要复盘，不然不知道为什么成功。"小想不解："失败了才复盘，成功有什么可复盘的？"

老李摇头："三种复盘，三种价值。遗漏复盘找系统性漏洞，翻车复盘找根因类型，成功复盘找可复制方法。"

**问题揭示**：需求遗漏、上线翻车、项目成功——三种场景都需要复盘。很多人只复盘失败，不复盘遗漏，更不复盘成功。

---

## 二、三种复盘方法

### 什么是遗漏复盘？

**遗漏复盘**是对遗漏的需求进行系统性分析，找到遗漏发生的环节和原因，建立机制避免同类遗漏再发生。

**遗漏高发区**：

| 遗漏类型 | 高发环节 | 典型特征 |
|---------|---------|---------|
| 异常流程遗漏 | 需求梳理 | 只考虑正常路径，忽略异常分支 |
| 边界条件遗漏 | 需求细化 | 忽略数值边界、状态边界 |
| 数据权限遗漏 | 需求评审 | 忽略不同角色的数据可见性 |
| 状态回退遗漏 | 需求设计 | 只考虑正向流程，忽略回退场景 |

就像装修遗漏：水电改造时忘了留空调插座。复盘追问——设计阶段为什么没考虑？

### 什么是翻车复盘？

**翻车复盘**是对上线问题进行系统性分析，区分技术原因、流程原因和沟通原因。

关键区分：**直接原因**和**根因**。

- 直接原因：导致问题的直接因素（技术层面）
- 根因：导致直接原因的系统性因素（流程或沟通层面）

小想的积分清零：直接原因是迁移脚本用旧规则，根因是变更通知机制缺失——沟通原因。

就像交通事故：直接原因是超速，根因可能是限速标志不明显。

### 什么是成功复盘？

**成功复盘**是对成功项目进行系统性分析，提炼可复制方法，识别偶然因素，避免"幸存者偏差"。

成功复盘的价值：提炼可复制方法、识别偶然因素、避免"幸存者偏差"。

就像冠军复盘：分析训练方法（可复制）、对手状态（偶然）、天气条件（运气）。

### 产品经理容易误解什么？

**误解一："复盘就是开会说一下"** — 正确理解：三种复盘都有明确流程和模板。

**误解二："翻车都是开发的问题"** — 正确理解：很多翻车根因在需求阶段——遗漏、模糊、变更管理不当。

**误解三："成功了就不需要复盘"** — 正确理解：成功复盘识别可复制方法vs偶然运气。

## 三、开发怎么看复盘？

开发看复盘，想的不是"谁的责任"，而是：

- 遗漏发生在哪个环节？需求、评审、开发、测试？
- 技术问题是代码问题还是流程问题？
- 成功是因为代码质量还是其他因素？

老李问张开发："这次积分清零，根因是什么？"

张开发说："代码没问题，我按文档写的。问题在于文档是旧的。"

老李追问："那根因是沟通？"

张开发点头："文档变更没通知，这是变更管理流程问题。开发需要知道变更才能改代码，信息不对称导致问题。"

老李总结："技术问题往往是表象。你要追问为什么会有这个问题，找到系统性根因。"

**Blameless原则**：复盘不追责个人，只分析系统。每个人都坦诚分享信息，不怕被追责。

就像医疗事故复盘：不是追责医生，而是分析流程——核对环节缺失、信息传递不畅、检查清单不完整。

## 四、产品经理应该怎么做？

### 遗漏复盘三步法

**遗漏环节追溯**：列出遗漏项 → 回溯生命周期 → 统计分布 → 针对高发环节改进

**5 Whys分析**：对每个遗漏追问5层"为什么"，找到根因。

示例：注销遗漏 → 需求没提到 → 没考虑未完成订单 → 模板无前置检查项 → 模板不完善 → 补充模板

### 翻车复盘六步法

还原时间线 → 定位直接原因 → 5 Whys追溯根因 → 区分原因类型 → 制定措施 → 跟踪落地

### 成功复盘要素分解法

列出成功要素 → 区分可复制/偶然要素 → 固化可复制方法 → 记录偶然因素

要素类型：方法要素（固化）、资源要素（评估依赖）、时机要素（明确特殊性）、运气要素（明确成分）、人员要素（区分个人vs系统）

### AI辅助

AI生成遗漏检查清单初稿、补充遗漏场景建议、分析成功要素。

> 提示词：我有个用户注销功能，请列出可能遗漏的场景，包括异常流程、边界条件等。

AI生成的是"草稿"。遗漏高发区、根因判断、成功要素分类，需要你来补。

### Checklist

**遗漏复盘**：列出遗漏项 → 回溯环节 → 5 Whys → 补检查清单

**翻车复盘**：还原时间线 → 定位直接原因 → 追溯根因 → 区分类型 → 制定措施跟踪

**成功复盘**：列出要素 → 区分可复制/偶然 → 固化方法 → 记录偶然因素

## 五、一句话总结

回看周一的复盘会：小想的注销遗漏，通过遗漏复盘找到需求模板缺失前置检查项；积分清零问题，通过翻车复盘识别根因是沟通机制缺失；成功项目，通过成功复盘区分可复制方法与偶然运气。

**复盘公式：遗漏找环节，翻车找根因，成功找可复制。**
