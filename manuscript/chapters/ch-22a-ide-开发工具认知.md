# CH-22a IDE — 开发工具认知

## 1. 本章要解决什么问题

IDE 是什么？为什么建议产品经理也装一个？

很多产品经理看到开发屏幕上花花绿绿的界面，只觉得「挺酷的」，不知道那是干什么的。你不需要学会写代码，但你需要知道 IDE 是什么——就像你知道 Figma 是什么一样。

理解 IDE，你才能理解开发在哪个「工作台」上干活。协作时，你知道他们的工具边界在哪里。

## 2. 真实业务场景开场

周一上午，小想第一次走进开发团队的办公室。

路过一个开发的工位，她看到屏幕上花花绿绿的界面——左边是彩色代码，右边有黑底白字的窗口，底部有小图标和数字跳动。

小想好奇地问：「这是什么软件？挺酷的。」

开发笑了笑：「这是我的 IDE，写代码的地方。」

小想：「IDE？」

开发：「集成开发环境。左边是代码，颜色不一样是语法高亮；右边是终端，能跑命令；下面是 Git 状态。」

小想：「哦，原来不是记事本啊。」

开发：「记事本只能写字。这东西能写、能调试、能运行、还能管版本——是开发的「工作台」。」

小想点点头。她想起自己看设计稿都在 Figma，原来开发也有自己的「Figma」。

## 3. 核心概念解释

### IDE 是什么？

**IDE（Integrated Development Environment，集成开发环境）** 是开发写代码的核心工具。

它不是「写代码的记事本」，而是集成了多种能力的综合工作台：

- **代码编辑器**：语法高亮、代码补全、错误提示
- **调试器**：断点调试、变量查看、调用栈追踪
- **版本管理**：Git 集成，可视化提交历史
- **终端**：直接运行命令，无需切换窗口
- **AI 辅助**（新一代 IDE）：代码生成、智能补全、问题解答

### 为什么需要 IDE？

没有 IDE 之前，开发要来回切换窗口：编辑器写代码、命令行编译、浏览器看效果。

有 IDE 之后：所有操作在一个窗口完成。错误实时提示，Git 可视化，调试直接看变量值。

**核心价值**：整合工具，减少「切窗口」的摩擦。

### 类比：IDE 之于开发，就像 Figma 之于设计师

| Figma（设计师） | IDE（开发者） |
|----------------|---------------|
| 设计稿编辑 | 代码编辑 |
| 组件库 | 代码库/模板 |
| 协作评论 | Git 提交记录 |
| 版本历史 | Git 分支/历史 |
| 插件扩展 | IDE 插件市场 |
| 输出切图/样式 | 输出编译产物 |

产品经理如果不理解 IDE，就像不理解设计师用 Figma 干什么一样——你不知道他们在哪个「工作台」上产出内容，协作时就容易产生沟通障碍。

### VS Code、JetBrains、Cursor 有什么区别？

| IDE | 定位 | 核心特点 |
|-----|------|----------|
| **VS Code** | 免费、轻量 | 几秒打开，插件丰富，零成本 |
| **JetBrains** | 专业、收费 | 代码分析最强，开箱即用，$149/年 |
| **Cursor** | AI 原生 | AI 内置，理解整个代码库，一句话改多文件 |

**VS Code**：微软出品，免费开源。适合前端、Python 开发、学生学习。

**JetBrains**：专业厂商。适合 Java/Kotlin 企业级开发、大型项目。

**Cursor**：2024 年爆火的 AI IDE。AI 能理解整个项目，不只是当前文件。产品经理可用来写需求、画原型。

## 4. 为什么产品经理容易误解

### 误解：IDE 就是「写代码的记事本」

很多产品经理看到开发用 IDE，心想：不就是写代码的软件吗？和记事本差不多吧？

这个误解的问题在于：你没看到 IDE 的「集成」能力。

记事本只能「写字」，IDE 是「写字 + 调试 + 运行 + 管理 + AI 辅助」的综合台。

类比：
- 记事本 = 只能写字的白纸
- IDE = 有格子、有公式栏、有图表工具、能自动计算、能保存版本的 Excel

**不是「写」，是「写 + 做 + 管」。**

### 误解原因

产品经理不写代码，看不到「调试」「版本管理」功能。只看到开发在「敲字」，就以为那是「写字」工具。

就像只看到设计师在 Figma 里「画图」，没看到组件库、版本对比、协作评论——不知道 Figma 的「协作」功能，就觉得它是画图软件。

## 5. 开发视角如何看这个问题

开发看 IDE，想的不是「有个软件写代码」，而是：

- 代码怎么组织？文件树要清晰
- 调试怎么跑？断点、变量、调用栈
- 版本怎么管？Git 提交、分支、合并
- 命令怎么跑？终端集成，不用切窗口
- AI 怎么用？补全、生成、问题解答

这些才是开发每天打交道的东西。

开发选择 IDE，考虑的是：

- **语言支持**：这个 IDE 支持我用的语言吗？
- **性能**：大项目打开快不快？
- **调试能力**：调试功能强不强？
- **AI 集成**：AI 辅助好不好用？

产品经理知道这些，就能理解开发为什么说「这个 IDE 不适合这个项目」。

## 7. AI 可以如何辅助

### 在 Cursor 中用 AI

- **写需求文档**：描述需求，AI 生成结构化文档
- **画原型代码**：描述页面，AI 生成 HTML/CSS 代码
- **验证想法**：描述逻辑，AI 帮你推演实现方案

### 注意边界

AI 生成的是「草稿」，你需要：

- 检查格式是否符合团队规范
- 补充遗漏的业务细节
- 验证逻辑是否完整

老李说：「AI 是工具，判断力是你的。工具帮你提速，你把控质量。」
