# CH-23 域名 DNS 证书与 CDN — 网络基础设施

## 1. 痛点场景

"域名买了吗？DNS解析配好了吗？SSL证书呢？备案呢？"

周一上午，小想兴冲冲去找运维老王："我们系统开发完了，准备上线！"

老王连问了四个问题。小想只答上来一个："域名买了，上周就买了。"

其他三个？DNS解析？SSL证书？备案？小想一脸懵。

老王笑了："域名只是个名字，买了还不能用。得配DNS解析才能找到服务器，得配SSL证书才能保证安全，国内还得备案才能上线。"

小想恍然大悟：原来上线不只是"开发完就行"，还有这么多基础设施要准备。

**你是不是也遇到过？**

系统上线前运维问的那些东西——域名、DNS、证书、CDN——到底是什么？为什么买了域名还不能用？为什么同一个网站在不同地方访问速度不一样？本章就来回答这些问题。

---

## 2. 核心概念：域名、DNS、证书、CDN

### 2.1 域名和IP：人和机器的"语言差"

**域名是人类可读的网址。**

比如 `baidu.com`、`taobao.com`——一看就知道是哪个网站。

**IP地址是机器识别的地址。**

比如 `202.108.22.5`——一串数字，机器用它来找服务器。

问题是：人记不住IP地址，机器只认IP地址。这就需要一个"翻译官"——DNS。

### 2.2 DNS：互联网的电话簿

**DNS（域名系统）把域名翻译成IP地址。**

类比：DNS就像电话簿。

| 现实世界 | 互联网世界 |
|---------|-----------|
| 人名 | 域名（如 `www.baidu.com`） |
| 电话号码 | IP地址（如 `202.108.22.5`） |
| 电话簿 | DNS服务器 |

你想打电话给张三，查电话簿找到他的号码，然后拨打。浏览器也一样——输入域名，DNS服务器告诉它IP地址，然后用IP访问网站。

**典型误解纠正**：域名买了就能用？

错了。域名只是"名字"，买了还得配DNS解析（把名字指向IP）、配SSL证书（安全证明）、国内还要备案（政府审批）。

类比理解：
- 买了域名 = 注册了名字
- DNS解析 = 把名字写入电话簿
- SSL证书 = 拿到身份证
- 备案 = 获得营业许可

### 2.3 HTTPS证书：网站的身份证

**HTTPS证书是保证网站安全的加密凭证。**

三个作用：

| 作用 | 说明 | 对用户的影响 |
|------|------|-------------|
| 数据加密 | 传输过程加密，防止窃取 | 登录密码、支付信息不会被截获 |
| 身份验证 | 证明网站是真的，不是冒牌 | 防止钓鱼网站 |
| 浏览器信任 | 显示"安全"标识 | 没证书浏览器警告"不安全" |

新系统上线必须配证书。没证书，浏览器会显示警告，用户不敢用。

**证书还有有效期**。一般一年，到期前要续期。过期了浏览器又警告"不安全"。

### 2.4 CDN：快递驿站网络

**CDN（内容分发网络）让用户就近访问，加快速度。**

类比：CDN就像快递驿站网络。

| 现实世界 | 互联网世界 |
|---------|-----------|
| 总仓（总部仓库） | 源站服务器（原始服务器） |
| 快递驿站（各小区门口） | CDN边缘节点（就近服务器） |
| 用户就近取货 | 用户就近访问 |

没有驿站：用户在郊区，快递员要从市中心总仓跑很远，等很久。

有驿站：常见货物提前放到各小区驿站，用户下楼就能取。

CDN也一样：图片、视频、CSS、JS这些静态内容缓存到全国各地的边缘节点，用户访问最近的节点，不用跨越半个中国找源站。

### 2.5 带宽和延迟：水管粗细和长度

**带宽是水管粗细——能流多少水。**

服务器带宽100Mbps，就像水管直径100毫米，能流很多水。

**延迟是水管长度——水流多久到。**

用户在北京，服务器在广州，数据要跨越一千多公里，延迟就高。

**关键认知**：CDN解决的是延迟问题，不是带宽问题。

典型误解："网站慢是因为带宽不够"

可能是。也可能是延迟太高——水管够粗，但太长了。

类比理解：
- 加带宽 = 水管变粗（同时能流更多水）
- 加CDN = 水管变短（用户离水源更近）

## 3. 运维怎么看这个问题

运维老王问的那些问题，背后是他的工作清单：

**DNS配置**
- 域名解析是否指向正确的IP？
- 解析记录类型配对了吗？（A记录、CNAME记录）
- TTL设置合理吗？（缓存时间，太短频繁查询，太长改了不生效）

**SSL证书**
- 证书申请了吗？
- 证书类型合适吗？（DV/OV/EV，验证等级不同）
- 证书部署了吗？
- 有效期监控了吗？（到期前提醒续期）

**备案流程**
- 提交备案申请了吗？
- 审核进度跟踪了吗？（一般1-2周）
- 备案号拿到了吗？

**CDN配置**
- 哪些内容需要CDN？（图片、视频、静态资源）
- CDN节点覆盖范围够吗？
- 缓存策略设置合理吗？

运维不是刁难，是防御性工作。上线前没准备好，上线后网站打不开、浏览器警告"不安全"、用户投诉"网站慢"，都是事故。

**产品经理和运维的信息差**：产品看到的是"上线发布"，运维看到的是"基础设施是否就绪"。产品以为开发完就行，运维要确保域名能解析、证书有效、备案通过、CDN加速生效。

## 4. 怎么做：上线前基础设施清单

系统上线前，产品经理要确认这些基础设施是否就绪。

### Step 1：用AI生成准备清单

提示词示例：
> 请帮我生成新系统上线前的域名、DNS、证书、CDN准备事项清单。

AI会给一个框架：
```
上线前检查：
- 域名是否购买
- DNS解析是否配置
- SSL证书是否申请和部署
- 国内服务器是否备案
- CDN是否需要、是否配置
```

### Step 2：人工补具体信息

AI不知道你的具体情况，你要补：

| 事项 | AI给的框架 | 你要补的具体信息 |
|------|-----------|-----------------|
| 域名 | 域名是否购买 | 哪个域名服务商？域名具体是什么？ |
| DNS解析 | DNS解析是否配置 | 解析到哪个服务器IP？配置截图确认 |
| SSL证书 | 证书是否申请 | DV/OV/EV哪个类型？有效期到什么时候？ |
| 备案 | 是否备案 | 备案进度？预计什么时候通过？ |
| CDN | 是否需要 | 图片视频多吗？用户分布在哪？ |

### Step 3：上线前基础设施Checklist

**域名准备**
- [ ] 域名已购买
- [ ] 品牌相关域名已保护注册
- [ ] 域名服务商账号可访问

**DNS配置**
- [ ] DNS解析已配置
- [ ] 解析指向正确的服务器IP
- [ ] 配置截图已确认

**SSL证书**
- [ ] 证书已申请
- [ ] 证书已部署到服务器
- [ ] 证书有效期已记录
- [ ] 续期提醒已设置

**备案（国内服务器）**
- [ ] 备案申请已提交
- [ ] 审核进度已跟踪
- [ ] 备案号已获取

**CDN评估**
- [ ] 是否有大量图片、视频、静态资源？
- [ ] 用户是否分布在全国各地？
- [ ] CDN是否已配置？
- [ ] CDN缓存策略是否合理？

### Step 4：时间规划

| 事项 | 准备时间 | 建议启动时间 |
|------|---------|-------------|
| 域名购买 | 即时 | 开发中期 |
| DNS配置 | 1天 | 上线前1周 |
| SSL证书申请部署 | 1-3天 | 上线前1周 |
| 备案 | 1-2周 | **开发中期就要启动** |
| CDN配置 | 1-2天 | 上线前或上线后优化 |

**关键提醒**：备案要1-2周，不能等开发完才申请，否则上线会延期。

## 5. 一句话总结

**域名是名字，DNS是电话簿，证书是身份证，CDN是驿站网。**

上线前：域名→DNS→证书→备案→CDN，缺一不可。
