# CH-22 IDE、Git 认知与 SSH

> 开发工具与服务器基础

---

## 开发的屏幕上那些工具都是什么

周一上午，小想路过开发老李的工位。

看到他屏幕上花花绿绿的界面：左边是彩色的代码，右边有个框框显示文件列表，下面还有个黑黑的窗口在滚动文字。

小想好奇地问："这是什么软件？看起来好复杂。"

老李笑了："这是 IDE，集成开发环境。我写代码用的工具。"

小想："我以为写代码就是用记事本..."

老李摇摇头："记事本只能打字。IDE 是'作战指挥室'。代码编辑、运行调试、版本管理、终端命令，都在这里。就像你用 Figma 做设计，不会用记事本画 UI 吧？"

---

## IDE：开发版的 Figma

IDE 是集成开发环境。

集成了编辑器、编译器、调试器、版本控制等多种功能。

就像 Figma 集成了画图、图层管理、协作评论、版本历史。设计师不会用记事本画 UI。同理，开发者不会用记事本写代码。

### VS Code、JetBrains、Cursor 有什么区别？

| 特性 | VS Code | JetBrains | Cursor |
|------|---------|-----------|--------|
| **定位** | 轻量编辑器 | 全功能 IDE | AI 原生 IDE |
| **价格** | 免费 | 收费 | 免费 + Pro $20/月 |
| **AI 能力** | 插件支持 | 插件支持 | 内置深度集成 |
| **适用场景** | Web 开发 | Java 企业项目 | AI 辅助开发 |

选择建议：

- Web 开发选 VS Code。轻量、插件多。
- Java 项目选 JetBrains。深度语义理解。
- 想 AI 辅助选 Cursor。对话式编程。

**误解纠正**：IDE 不是"写代码的记事本"。IDE 集成了语法高亮、调试、版本管理、终端、AI 辅助等能力。是开发的核心武器。

## Git：需求变更的时光机

需求评审会上，老李问小想："上次那版需求文档你改了哪里？"

小想挠头："嗯...改了好几处。我记得有个支付流程改了，还有什么..."

老李："有没有记录？"

小想："我改完就保存了。没记录具体改了哪里..."

老李叹气："这就是为什么开发需要 Git。每次改动都记录：改了什么、谁改的、什么时候改的。"

**Git 是分布式版本控制系统。**

记录每次文件变更的历史。

就像时光机。可以回到任何历史版本，看到每次改了什么、谁改的、为什么改。

### 核心概念一览

| 术语 | 含义 | 类比 |
|------|------|------|
| **Commit** | 一次变更的快照 | 按一下"保存"，附带说明 |
| **Branch** | 独立的开发线 | 平行宇宙，安全做实验 |
| **Merge** | 把不同分支合在一起 | 把平行宇宙的结果带回主线 |
| **Conflict** | Git 无法自动决定保留哪个版本 | 两个宇宙改了同一行，需人工裁决 |

**误解纠正**：Git 不是程序员的专属工具。

版本控制的思维是通用的。需求文档也需要版本管理。设计稿也需要变更追溯。产品决策也需要历史记录。

理解 Git，你能更好地管理需求变更。与开发沟通更顺畅。理解发布节奏和版本规划。

## SSH：远程操作的遥控器

周五下午，产品上线出了问题。

老李说："我 SSH 到服务器看一下日志。"

小想困惑："SSH？是一种加密算法吗？"

老李笑了："SSH 是 Secure Shell。一种远程登录协议。就像用遥控器远程控制电视。SSH 让我不用去机房就能操作服务器。"

小想："那'看日志'是什么意思？"

老李："日志就是服务器运行的'黑匣子'。记录所有操作。我 SSH 登上去，查看日志文件，就能知道哪里出错了。"

十分钟后，老李找到了问题。

"原来是支付接口超时。日志里显示 500 错误。"

**SSH 是安全远程连接协议。**

用于登录服务器并执行命令。

运维说"SSH 到服务器看日志"，意思是：远程登录服务器，查看系统运行记录，找出问题原因。

**误解纠正**：SSH 不是加密算法。

SSH 是 Secure Shell，一种远程登录协议。加密只是手段，保证传输安全。实际用途是远程执行命令。

## 开发怎么看这些工具

开发看 IDE，想的不是"有个地方打字"。

而是：

- 语法高亮能帮我快速识别错误
- 调试器能让我逐行排查问题
- Git 集成能让我方便管理版本
- 终端能让我直接执行命令

开发看 Git，想的不是"有几个版本"。

而是：

- 状态怎么存储？
- 跳转条件怎么判断？
- 并发时状态冲突怎么办？
- 每次提交有没有清晰说明？

开发看 SSH，想的不是"远程登录"。

而是：

- 日志文件在哪里？
- 用什么命令查看？
- 错误信息怎么解读？
- 怎样快速定位问题？

## 产品经理应该怎么做

### 装一个 Cursor 体验 AI 协作

产品经理不需要成为开发者。

但可以装一个 Cursor，体验 AI 辅助的工作流。

就像设计师装 Figma 体验设计工作流。不需要成为专业开发者，但能理解工具的角色。

**Cursor 可以帮你做什么？**

1. **写需求文档**：描述业务流程，AI 生成结构化初稿。
2. **理解技术逻辑**：问"这个代码库的用户登录是怎么实现的"，AI 用通俗语言解释。
3. **做原型验证**：让 AI 生成可运行的 HTML，直接在浏览器看效果。

### 了解 Git 思维管理变更

不需要学会 Git 命令。

但要理解版本控制的思维：

- 每次变更有记录（谁改、何时、为什么）
- 可以对比任何两个版本
- 可以回退到任何历史版本
- 可以并行开发（分支）

下次改需求文档时，问自己：

- 这次改了什么？
- 和上次有什么区别？
- 为什么要改？

这就是 Git 思维。
