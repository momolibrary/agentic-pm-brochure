# CH-37 Human+Agent 协作模型 — 未来披萨团队与超级个体协作

---

## 本章要解决什么问题

AI 时代，团队协作方式正在被重塑。一个人类 + 多个 AI Agent，能否覆盖传统需要多人协作的业务域？产品经理需要理解这个趋势，为未来的工作方式做准备。

---

## 真实业务场景开场

周一早晨，产品团队例会。

小想兴奋地举起手机：「你们看这条新闻——硅谷出现第一个一人独角兽公司！创始人 Sarah，一个人加她的 AI Agent 团队，做到了 10 亿美金估值。」

老李点点头：「我看过这个案例。Sarah 的「团队」是这样：她自己负责战略决策、品牌定位、关键客户关系，每天只工作 2 小时。剩下的事——内容生成、数据分析、客服响应、销售跟进——全部交给 AI Agent。」

小想瞪大眼睛：「一个人...干 50 个人的活？」

「不是一个人干 50 个人的活。」老李纠正，「是 Sarah 的判断力，被 Agent 放大了 50 倍。她做判断，Agent 去执行。」

这个故事引发了更深层的问题：未来的团队会变成什么样？一个产品经理 + AI Agent，能覆盖一个业务域吗？

## 核心概念解释

### 什么是「超级个体」？

**超级个体** = 一个人类 + 多个 AI Agent 组成的协作单元。

它能覆盖传统需要多人协作的业务域。核心分工：
- Human 提供**判断力**（战略决策、伦理考量、关系维护）
- Agent 提供**执行力**（内容生成、数据分析、流程自动化）

### 类比一：Human = 指挥家，Agent = 乐团

指挥家不演奏乐器。但他决定演奏什么、怎么演奏、节奏快慢。

乐团成员负责执行。他们技术娴熟、24/7 可用，各有专长（弦乐、打击、管乐）。

没有指挥家，乐团只是一堆乐器，无法协调。没有乐团，指挥家只是空有想法，无法产出。

Sarah 就像指挥家。她每天 2 小时做指挥工作——决定产品方向、审核 Agent 产出。她的 Agent 团队是乐团，24/7 执行。

### 类比二：超级个体 = CEO，Agent = 各职能部门

CEO 不做具体业务。但他掌握战略方向、品牌调性、关键决策。

各职能部门负责执行：市场部做品牌传播，销售部跟进线索，客服部处理投诉，数据部做分析报告。

一个 CEO + 各部门 = 一家公司。一个超级个体 + 各 Agent = 一个「一人公司」。

Sarah 的 Agent-A 是市场部（内容生成），Agent-B 是数据部（市场分析），Agent-C 是客服部（自动响应），Agent-D 是销售部（线索跟进）。她是 CEO，只做判断。

### 为什么「披萨团队」会演变？

Amazon 的两披萨团队规则：团队不超过两张披萨能喂饱的人数（6-8 人）。

为什么？**沟通成本定律**：

```
沟通通道数 = n(n-1)/2
```

| 团队规模 | 沟通通道数 |
|---------|-----------|
| 2 人 | 1 |
| 5 人 | 10 |
| 8 人 | 28 |
| 10 人 | 45 |
| 20 人 | 190 |

团队越大，沟通成本随人数平方增长。Brooks 定律说得更直白：**向延迟的项目添加人力，只会让它更延迟**。因为新人需要融入时间，沟通成本爆炸。

Human+Agent 模式改变了这个公式：

```
传统披萨团队（2010年代）
  6-8 人，沟通通道 15-28 条
  全靠 Human 沟通

Human+Agent 披萨小组（2020年代中）
  2-3 人 + 各配 Agent
  Human 间沟通通道只有 3 条
  覆盖能力 ≈ 传统 6-8 人团队

超级个体小组（2030年代）
  1-2 人 + Agent 团队
  Human 间沟通通道只有 1 条
  产出 ≈ 传统 10-20 人团队
```

披萨团队的规则没消失——核心仍是**控制沟通复杂度**。但手段变了：用 Agent 替代 Human，减少「肉脑袋间」的沟通。

## 为什么产品经理容易误解

### 误解一：「AI 替代人」

听到一人独角兽，很多人第一反应是：AI 要替代人了。

**误解在哪**：你没看到「放大」的本质。

Sarah 不是被 AI 替代。相反，她的能力被放大了：
- 以前她要管理 50 人团队，每天开会、协调、处理人际问题
- 现在她只需要做判断，执行交给 Agent
- 她的判断力变成产品，Agent 把判断变成执行结果

AI 不替代判断力。AI 替代的是重复执行。Human 判断 + Agent 执行 = 放大，不是替代。

### 误解二：「团队人越多效率越高」

很多产品经理遇到瓶颈，本能反应是「加人」。

**误解在哪**：你没算过沟通成本。

团队从 8 人扩展到 20 人，沟通通道从 28 条变成 190 条。6 倍的沟通成本，换来的是更多人需要协调。

Human+Agent 模式的答案：精简 Human + 补充 Agent。沟通成本可控，执行力放大。

## 开发视角如何看这个问题

开发看 Human+Agent 协作，关注的是**Agent 如何落地**。

### Agent 是「合成员工」

从技术视角，Agent 是一段代码。但从协作视角，Agent 是「合成员工」：
- 有明确的职责边界
- 有可定义的任务输入/输出
- 有可衡量的执行质量

开发要回答的问题是：这个「合成员工」能做什么、不能做什么？边界在哪？

### Agent 编排框架选择（2025）

| 框架 | 适用场景 | 特点 |
|------|---------|------|
| CrewAI | 角色型 Agent 团队，快速原型 | 简单易学，适合内容创作、客服 |
| LangGraph | 复杂状态流，生产级系统 | 状态管理强，适合审批流程、多步推理 |
| AutoGen | 会话型 Agent，研究实验 | 会话模式强，适合代码审核、协作推理 |

选择建议：快速开始用 CrewAI，生产级复杂流用 LangGraph。

### 技术边界提醒

Agent 能做的：
- 规则明确的重复任务
- 数据处理和报告生成
- 基于模板的内容创作

Agent 做不了的：
- 需要人类判断的战略决策
- 需要情感理解的客户关系
- 需要伦理考量的边界问题

## 产品经理应该如何做

### Human/Agent 分工边界判断

问自己三个问题：

1. **这件事需要判断力吗？**
   - 如果需要权衡多方利益、做战略取舍 → Human 做
   - 如果有明确规则、可模板化 → Agent 做

2. **这件事能定义输入/输出吗？**
   - 如果能明确描述输入是什么、输出是什么 → Agent 可以做
   - 如果是开放式探索、需要现场判断 → Human 做

3. **这件事出错后果有多严重？**
   - 如果出错可容错、可回滚 → Agent 可以试
   - 如果出错影响关键客户、品牌声誉 → Human 审核

### Human+Agent 组间沟通方式

传统团队沟通靠开会。Human+Agent 组沟通走**精准接口**：

| 传统方式 | Human+Agent 方式 |
|---------|-----------------|
| 发邮件约会议时间 | 创建 Issue，描述变更内容 |
| 开会讨论 30 分钟 | Agent 自动解析需求，生成任务 |
| 会议纪要整理 | 另一组 Agent 接收任务，开始执行 |
| 邮件确认纪要 | Human 只需审核结果 |
| 信息遗漏，再开会 | 所有信息在 Issue 中可追溯 |

**类比**：传统沟通是「打电话」，必须双方同时在线。Human+Agent 组沟通是「API 调用」，异步高效。

**原则**：能用文档/Issue/API 解决的，别开会。

### AI 可以如何辅助

- 用 AI 生成 Agent 任务配置初稿
- 用 AI 做 Agent 执行结果的质量初筛
- 用 AI 监控 Agent 运行状态，异常预警

但记住：**Agent 配置是草稿，判断力是你的**。Human 审核 Agent 产出，这是不可替代的环节。

## 案例复盘

回看 Sarah 的一人独角兽案例：

Sarah 不是超人。她的判断力是稀缺资源，每天只能投入 2 小时。但 Agent 把这 2 小时的判断，放大成 24/7 的执行力。

她做到了传统 50 人团队的产出。但她没有管理 50 人。她只管理判断，Agent 管理执行。

关键门槛：
- 判断力稀缺（很多人只会执行，不会判断）
- Agent 编排能力（这是新技能，需要学习）
- 信任边界（敢把重要工作交给 Agent）

## 本章总结

Human 判断 Agent 执行，API 化沟通减少会议。超级个体 = 判断力放大器，不是人被替代。
